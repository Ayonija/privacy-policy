# FAANG Roadmap — The Complete 6-Month Preparation Plan

> **Goal:** Master DSA, practise timed assessments, refine system design, and build interview confidence.

---

| Area | Time | Focus |
|---|---|---|
| DSA (Data Structures & Algorithms) | 60% | Patterns over quantity |
| System Design | 30% | Scalability, Caching, DB Sharding |

---

## 6-Month Master Plan

### Months 1–3 — DSA Foundations

| Focus | Details |
|---|---|
| Progression | Easy → Medium → Hard |
| Daily target | 5–10 problems per day |
| Measure of success | Speed + accuracy, not just completion |

**What to do:**
- Follow a structured sheet (Striver's SDE Sheet or NeetCode Roadmap)
- Solve by pattern, not by topic order alone
- After each problem: state time complexity, space complexity, and at least one alternative approach
- Weekly review: revisit problems you couldn't solve in under 20 minutes

---

### Month 4 — Exposure & Assessment

| Activity | Purpose |
|---|---|
| Connect with mentors / seniors | Understand real interview processes, common pitfalls |
| Screening mock tests | Simulate actual OA pressure |
| LeetCode contests | Timed problem-solving, leaderboard ranking |
| HackerRank timed tests | OA format practice |

**Weekly structure:**
- 3 days: continue DSA (medium–hard)
- 1 day: LeetCode/HackerRank timed contest
- 1 day: mentor call or recorded interview debrief
- Weekend: review contest problems, note patterns missed

---

### Month 5 — Mock Interviews & Communication

| Activity | Purpose |
|---|---|
| Structured mock interviews | Simulate full interview loop |
| Daily DSA (maintained) | Keep problem-solving sharp |
| Communication practice | Narrate approach before coding; explain trade-offs |
| Confidence building | Repetition under observation removes anxiety |

**Drill format for each mock:**
1. Read problem aloud
2. Classify pattern (30 sec)
3. State brute force + complexity
4. Optimize, narrate approach
5. Code clean solution
6. Trace through 2 edge cases

---

### Month 6 — Final Sprint

| Week | Focus |
|---|---|
| Week 1 | Full DSA revision — every pattern, every template |
| Week 2 | Revision + 2 Online Assessment tests (timed, simulated) |
| Week 3–4 | Intensive mock interviews + full interview simulations |

**Week 3–4 daily schedule:**
- Morning (1.5 h): 2 medium + 1 hard LeetCode (timed)
- Afternoon (1 h): System design mock or HLD review
- Evening (30 min): Behavioral question using STAR + Leadership Principles

## Pattern Priority Cheatsheet

| Pattern | Frequency | When to use |
|---|---|---|
| Sliding Window | Very High | Subarray/substring with constraint |
| Two Pointers | Very High | Sorted array, pair/triplet problems |
| Fast / Slow Pointers | High | Cycle detection, middle of list |
| BFS / DFS | Very High | Trees, graphs, shortest path |
| Binary Search | High | Sorted input, answer-space search |
| Topological Sort | Medium | Dependency ordering, DAG |
| Heap (Top-K) | High | K largest/smallest, streaming median |
| DP | High | Optimal substructure, overlapping subproblems |
| Backtracking | Medium | Subsets, permutations, constraint satisfaction |
| Union-Find | Medium | Connected components, cycle detection in graph |
| Monotonic Stack | Medium | Next greater/smaller element |
| Trie | Medium | Prefix search, autocomplete |

## Interview Day Checklist

- [ ] Classify pattern in first 2 minutes (use Pattern Map habit)
- [ ] State brute force + complexity before optimizing
- [ ] Talk through edge cases before writing: empty input, single element, duplicates, negatives, overflow
- [ ] Narrate approach — interviewers evaluate communication, not just code
- [ ] If stuck: name the pattern you think applies and explain why
- [ ] For L5+: discuss alternative approaches and trade-offs unprompted
