## DSA (1.5 hours — OA Contest Mode)

### Pattern: Min-Heap Enumeration of Ranked Sums / Combinations
- Core idea: When you need the Kth best combination from multiple sorted arrays, seed a min-heap with the best starting state and expand outward by pushing neighboring states (swap one element, extend by one element). Each pop gives the next best combination in order. This avoids generating all combinations explicitly.
- Trigger condition: Use heap enumeration when the problem asks for the Kth smallest/largest sum, product, or combination from sorted inputs, and enumerating all possibilities is too slow (O(N^K) brute force).
- Time complexity: O(N log N + K log K) | Space complexity: O(K) heap

### Contest Template (write from memory before solving):
```java
// Min-heap enumeration — Kth smallest sum pattern
// Seed: best possible state (index 0 of sorted structure)
// Expand: from state (value, index), push (value + next_element, next_index)
//          and optionally (value - current + next, next_index) for swap variant

PriorityQueue<long[]> heap = new PriorityQueue<>(Comparator.comparingLong(a -> a[0]));
heap.offer(new long[]{seed_value, seed_index});

for (int k = 0; k < K - 1; k++) {
    long[] top = heap.poll();
    long val = top[0];
    int idx  = (int) top[1];
    // push extend: add next element
    if (idx + 1 < sortedArr.length)
        heap.offer(new long[]{val + sortedArr[idx + 1], idx + 1});
    // push swap: replace current element with next (for K-sum variant)
    if (idx > 0)
        heap.offer(new long[]{val - sortedArr[idx] + sortedArr[idx + 1], idx + 1});
}
return heap.peek()[0]; // Kth value
```

### Problems

| # | Problem | LC # | Difficulty | Pattern | Key Insight |
|---|---------|------|------------|---------|-------------|
| 1 | Find the K-Sum of an Array | 2386 | Hard (Cold Solve) | Min-Heap Enumeration on Sorted Absolute Values | Max sum = sum of all positives; replace each num with abs(num); sort descending; the Kth largest sum = maxSum minus the (K-1)th smallest reduction; use min-heap of (reduction, index); each pop: push (reduction+sorted[i+1], i+1) to extend, and (reduction-sorted[i]+sorted[i+1], i+1) to swap |
| 2 | Find K Pairs with Smallest Sums | 373 | Medium (Revision) | Min-Heap with (sum, i, j) Tuples | REVISION: Seed min-heap with (nums1[i]+nums2[0], i, 0) for all i in nums1; each pop of (sum, i, j): record pair [nums1[i], nums2[j]]; push (nums1[i]+nums2[j+1], i, j+1) if j+1 < nums2.length; stop after K pairs |
| 3 | Longest Happy String | 1405 | Medium (OA Timed) | Max-Heap Greedy Character Selection | OA: Set 20-min timer. Max-heap ordered by character count; greedily pick the most frequent character; if the last two characters already equal the most frequent, pick the second most frequent instead; stop when no valid character can be chosen; append chosen character and re-push with count-1 |

### Code Skeleton

```java
class Solution {
    // LC 2386 — Find the K-Sum of an Array
    // (cold solve — 30-min timer, narrate before coding)
    // TODO: public long kSum(int[] nums, int k)
    // Key: Compute maxSum = sum of all positive values. Convert each num to abs(num),
    //      sort ascending. The problem reduces to: find the (k-1)th smallest subset sum
    //      of the absolute-value array (the "reduction" from maxSum). Use a min-heap
    //      of (reduction, index). Seed with (sorted[0], 0) — the smallest possible reduction.
    //      Each pop (r, i): push (r+sorted[i+1], i+1) [extend: add next element to subset]
    //                  and  (r-sorted[i]+sorted[i+1], i+1) [swap: replace sorted[i] with sorted[i+1]].
    //      After k-1 pops, the heap top's reduction value subtracted from maxSum gives the kth largest sum.

    // LC 373 — Find K Pairs with Smallest Sums (revision — write from memory)
    public List<List<Integer>> kSmallestPairs(int[] nums1, int[] nums2, int k) {
        List<List<Integer>> result = new ArrayList<>();
        if (nums1.length == 0 || nums2.length == 0) return result;
        PriorityQueue<int[]> heap = new PriorityQueue<>(Comparator.comparingInt(a -> a[0]));
        for (int i = 0; i < Math.min(nums1.length, k); i++)
            heap.offer(new int[]{nums1[i] + nums2[0], i, 0});
        while (!heap.isEmpty() && result.size() < k) {
            int[] top = heap.poll();
            int i = top[1], j = top[2];
            result.add(Arrays.asList(nums1[i], nums2[j]));
            if (j + 1 < nums2.length)
                heap.offer(new int[]{nums1[i] + nums2[j + 1], i, j + 1});
        }
        return result;
    }

    // LC 1405 — Longest Happy String
    // (OA simulation — set 20-min timer before reading this problem)
    // TODO: public String longestDiverseString(int a, int b, int c)
    // Key (unlock after 20 min if stuck): Use a max-heap of int[]{count, charCode}.
    //      Each step: poll the most frequent char. If the last TWO characters of the
    //      result are already this char, poll the SECOND most frequent instead.
    //      Append chosen char, decrement count, re-offer if count > 0.
    //      Re-offer the skipped char back if you had to pick the second-most frequent.
    //      Stop when heap is empty or no valid char exists (only one char type remains
    //      and its last two are already appended).
}
```

## System Design (30 min — LLD Deep Dive)

### Topic: Movie Ticket Booking DB Schema (BookMyShow)

#### Schema

```sql
-- Movies
Movies (
    movie_id      BIGINT       PRIMARY KEY AUTO_INCREMENT,
    title         VARCHAR(255) NOT NULL,
    duration_mins INT          NOT NULL,
    genre         VARCHAR(100),
    language      VARCHAR(50),
    rating        DECIMAL(3,1)
)

-- Theaters
Theaters (
    theater_id BIGINT       PRIMARY KEY AUTO_INCREMENT,
    name       VARCHAR(255) NOT NULL,
    city       VARCHAR(100) NOT NULL,
    address    TEXT
)

-- Screens (a theater has many screens)
Screens (
    screen_id    BIGINT       PRIMARY KEY AUTO_INCREMENT,
    theater_id   BIGINT       NOT NULL REFERENCES Theaters(theater_id),
    screen_name  VARCHAR(50)  NOT NULL,
    total_seats  INT          NOT NULL
)

-- Showtimes (a screen shows a movie at a specific time)
Showtimes (
    showtime_id BIGINT          PRIMARY KEY AUTO_INCREMENT,
    movie_id    BIGINT          NOT NULL REFERENCES Movies(movie_id),
    screen_id   BIGINT          NOT NULL REFERENCES Screens(screen_id),
    start_time  DATETIME        NOT NULL,
    end_time    DATETIME        NOT NULL,
    base_price  DECIMAL(8,2)    NOT NULL
)

-- Seats (physical seats in a screen — static, not per-showtime)
Seats (
    seat_id     BIGINT       PRIMARY KEY AUTO_INCREMENT,
    screen_id   BIGINT       NOT NULL REFERENCES Screens(screen_id),
    row_label   CHAR(1)      NOT NULL,  -- e.g., 'A', 'B', 'C'
    seat_number INT          NOT NULL,
    type        ENUM('REGULAR', 'PREMIUM', 'RECLINER') NOT NULL DEFAULT 'REGULAR'
)

-- Bookings (one booking per user per showtime, may cover multiple seats)
Bookings (
    booking_id   BIGINT          PRIMARY KEY AUTO_INCREMENT,
    user_id      BIGINT          NOT NULL,
    showtime_id  BIGINT          NOT NULL REFERENCES Showtimes(showtime_id),
    total_amount DECIMAL(10,2)   NOT NULL,
    status       ENUM('LOCKED', 'CONFIRMED', 'CANCELLED') NOT NULL DEFAULT 'LOCKED',
    expires_at   DATETIME        NOT NULL  -- LOCKED seats released after this
)

-- BookingSeats (junction: one row per seat per booking)
BookingSeats (
    booking_id   BIGINT NOT NULL REFERENCES Bookings(booking_id),
    seat_id      BIGINT NOT NULL REFERENCES Seats(seat_id),
    showtime_id  BIGINT NOT NULL REFERENCES Showtimes(showtime_id),
    PRIMARY KEY (seat_id, showtime_id)    -- uniqueness constraint: one booking per seat per showtime
)
```

#### Seat Reservation Flow
```
1. User selects seats → application sends seat_ids + showtime_id
2. BEGIN TRANSACTION
3. INSERT INTO Bookings (status='LOCKED', expires_at=NOW()+INTERVAL 10 MINUTE)
4. INSERT INTO BookingSeats (booking_id, seat_id, showtime_id)
   — if duplicate key on (seat_id, showtime_id): ROLLBACK → return "seat no longer available"
5. COMMIT → return booking_id to client
6. User completes payment
7. UPDATE Bookings SET status='CONFIRMED' WHERE booking_id=?
8. Background job (runs every 1 min):
   DELETE FROM BookingSeats WHERE booking_id IN (
       SELECT booking_id FROM Bookings WHERE status='LOCKED' AND expires_at < NOW()
   )
   UPDATE Bookings SET status='CANCELLED' WHERE status='LOCKED' AND expires_at < NOW()
```

#### Key Indexes
```sql
CREATE INDEX idx_showtimes_movie   ON Showtimes(movie_id, start_time);
CREATE INDEX idx_showtimes_screen  ON Showtimes(screen_id, start_time);
CREATE INDEX idx_bookings_user     ON Bookings(user_id);
CREATE INDEX idx_bookings_showtime ON Bookings(showtime_id, status);
CREATE INDEX idx_seats_screen      ON Seats(screen_id);
```

**Interview talking point:** "If asked how BookMyShow prevents two users from booking the same seat, answer: the PRIMARY KEY on (seat_id, showtime_id) in BookingSeats acts as a uniqueness constraint — the second INSERT fails with a duplicate key error, which the application catches to roll back the transaction and show the user 'seat no longer available'. No explicit SELECT-then-INSERT race condition is possible because the constraint is enforced atomically by the database engine."

## Flashcards

| Q | A |
|---|---|
| What are the two heap pushes in LC 2386's reduction enumeration and what does each represent? | Extend push: (reduction + sorted[i+1], i+1) — adds the next element to the current subset, increasing the reduction from maxSum. Swap push: (reduction - sorted[i] + sorted[i+1], i+1) — replaces sorted[i] with sorted[i+1] in the subset, keeping subset size the same but swapping to the next element. |
| In LC 373, why do you only seed the heap with (nums1[i]+nums2[0]) and not all (i, j) pairs? | nums2[0] is the smallest element in nums2, so (nums1[i], nums2[0]) gives the smallest possible sum for each fixed i. All other pairs (nums1[i], nums2[j]) with j>0 can be reached by expanding from (i, 0) via the heap, ensuring we enumerate pairs in ascending sum order without redundancy. |
| What is the invariant that makes the PRIMARY KEY on (seat_id, showtime_id) sufficient to prevent double-booking? | The database engine enforces uniqueness atomically — two concurrent INSERTs with the same (seat_id, showtime_id) cannot both succeed; exactly one will get a duplicate key error, causing the losing transaction to roll back. This requires no application-level locking. |
| What is the greedy invariant maintained in LC 1405 (Longest Happy String)? | At each step, always pick the most frequent available character unless the last two characters in the result are already that character. In that case, pick the second most frequent. This greedy choice maximizes string length because not using the most frequent character now would waste it — it must be used as soon as possible. |
| What does the expires_at column in the Bookings table accomplish in the BookMyShow schema? | It implements a time-boxed seat lock: a seat reserved (status=LOCKED) must be paid for before expires_at; if not, a background job releases the lock by deleting from BookingSeats and marking the booking CANCELLED. This prevents seats from being held indefinitely by users who abandon checkout. |
