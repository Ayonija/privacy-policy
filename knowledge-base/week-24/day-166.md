## DSA (1.5 hours — OA Contest Mode)

### Pattern: Sliding Window on Positions (Collect → Prefix Sum → Median Cost)
- Core idea: When a problem asks to bring K elements together with minimum adjacent swaps, collect the positions of those elements into array P, then slide a window of size K over P. The optimal meeting point is the median of the window; cost equals sum of absolute distances to the median, computable in O(1) per window via prefix sums.
- Trigger condition: "minimum adjacent swaps to make K elements consecutive/contiguous" or any cost-to-gather problem on a 0/1 array.
- Time complexity: O(N) | Space complexity: O(N)

### Contest Template (write from memory before solving):
```java
// Sliding window on collected positions — median cost via prefix sums
int[] P = new int[countOfTargets];
int pi = 0;
for (int i = 0; i < n; i++) if (arr[i] == target) P[pi++] = i;

long[] pre = new long[P.length + 1];
for (int i = 0; i < P.length; i++) pre[i + 1] = pre[i] + P[i];

long ans = Long.MAX_VALUE;
int K = P.length; // or given K
for (int r = K - 1; r < P.length; r++) {
    int l = r - K + 1;
    int mid = (l + r) / 2;
    // cost left half: P[mid]*[mid-l+1] - sum(P[l..mid])
    long leftCost  = P[mid] * (long)(mid - l + 1) - (pre[mid + 1] - pre[l]);
    // cost right half: sum(P[mid+1..r]) - P[mid]*(r-mid)
    long rightCost = (pre[r + 1] - pre[mid + 1]) - P[mid] * (long)(r - mid);
    // subtract offset: gaps between positions aren't real swap steps
    long offset = (pre[r + 1] - pre[l]) - ((long)P[l] * K + K * (K - 1) / 2);
    // (offset formula differs per problem — verify)
    ans = Math.min(ans, leftCost + rightCost - offset);
}
```

### Code Skeleton

```java
class Solution {

    // LC 1703 — Minimum Adjacent Swaps for K Consecutive Ones
    // (cold solve — 30-min timer, narrate approach before opening IDE)
    // TODO: public long minMoves(int[] nums, int k)
    // Key: collect positions of 1s; window of K; median minimizes L1 cost;
    //      prefix sums give window cost in O(1); subtract offset = positional gaps
    //      that don't require real swaps (P[l]*K + K*(K-1)/2 term).
    public long minMoves(int[] nums, int k) {
        List<Long> pos = new ArrayList<>();
        for (int i = 0; i < nums.length; i++) if (nums[i] == 1) pos.add((long) i);
        int m = pos.size();
        long[] pre = new long[m + 1];
        for (int i = 0; i < m; i++) pre[i + 1] = pre[i] + pos.get(i);

        long ans = Long.MAX_VALUE;
        for (int r = k - 1; r < m; r++) {
            int l = r - k + 1;
            int mid = (l + r) / 2;
            long pMid = pos.get(mid);
            // left half cost
            long lCost = pMid * (mid - l + 1) - (pre[mid + 1] - pre[l]);
            // right half cost
            long rCost = (pre[r + 1] - pre[mid + 1]) - pMid * (r - mid);
            long rawCost = lCost + rCost;
            // subtract gaps (positions aren't 0-indexed consecutive)
            long offset = (pre[r + 1] - pre[l]) - (pos.get(l) * k + (long) k * (k - 1) / 2);
            ans = Math.min(ans, rawCost - offset);
        }
        return ans;
    }

    // LC 567 — Permutation in String (revision — write from memory)
    public boolean checkInclusion(String s1, String s2) {
        if (s1.length() > s2.length()) return false;
        int[] diff = new int[26];
        for (char c : s1.toCharArray()) diff[c - 'a']++;
        for (char c : s2.substring(0, s1.length()).toCharArray()) diff[c - 'a']--;
        int nonZero = 0;
        for (int v : diff) if (v != 0) nonZero++;
        if (nonZero == 0) return true;
        for (int i = s1.length(); i < s2.length(); i++) {
            // add right
            int r = s2.charAt(i) - 'a';
            if (diff[r] == 0) nonZero++;
            diff[r]--;
            if (diff[r] == 0) nonZero--;
            // remove left
            int l = s2.charAt(i - s1.length()) - 'a';
            if (diff[l] == 0) nonZero++;
            diff[l]++;
            if (diff[l] == 0) nonZero--;
            if (nonZero == 0) return true;
        }
        return false;
    }

    // LC 1695 — Maximum Erasure Value (OA simulation — set 20-min timer before reading)
    // TODO: public int maximumUniqueSubarray(int[] nums)
    // Key (unlock after 20 min if stuck): HashSet tracks current window elements;
    //     shrink left until nums[right] can be added; maintain running sum.
    public int maximumUniqueSubarray(int[] nums) {
        Set<Integer> seen = new HashSet<>();
        int left = 0, sum = 0, best = 0;
        for (int right = 0; right < nums.length; right++) {
            while (seen.contains(nums[right])) {
                seen.remove(nums[left]);
                sum -= nums[left++];
            }
            seen.add(nums[right]);
            sum += nums[right];
            best = Math.max(best, sum);
        }
        return best;
    }
}
```

## System Design (30 min — LLD Deep Dive)

### Topic: LRU Cache + LFU Cache Class Design

#### LRU Cache — O(1) get/put

```
Class: LRUCache
Fields:
  - int capacity
  - int size
  - Map<Integer, DLinkedNode> map          // key → node
  - DLinkedNode head                        // sentinel (MRU side)
  - DLinkedNode tail                        // sentinel (LRU side)

Inner Class: DLinkedNode
  - int key
  - int val
  - DLinkedNode prev
  - DLinkedNode next

Methods:
  + LRUCache(int capacity)
  + int get(int key)
      // if absent: return -1
      // move node to front (after head sentinel); return node.val
  + void put(int key, int value)
      // if exists: update val + moveToFront(node)
      // else: create node + addToFront(node) + map.put(key, node) + size++
      //   if size > capacity: remove tail.prev from map + list; size--

Private helpers:
  - void addToFront(DLinkedNode node)       // insert after head sentinel
  - void removeNode(DLinkedNode node)       // re-link prev/next around node
  - void moveToFront(DLinkedNode node)      // removeNode + addToFront
```

#### LFU Cache — O(1) get/put

```
Class: LFUCache
Fields:
  - int capacity
  - int minFreq
  - Map<Integer, Integer> vals             // key → value
  - Map<Integer, Integer> freqs            // key → frequency
  - Map<Integer, LinkedHashSet<Integer>> freqBuckets
    // freq → ordered set of keys (insertion order = LRU order within freq)

Methods:
  + LFUCache(int capacity)
  + int get(int key)
      // if absent: return -1
      // incrementFreq(key); return vals.get(key)
  + void put(int key, int value)
      // if exists: vals.put(key, value); incrementFreq(key); return
      // if capacity == 0: return
      // if vals.size() == capacity: evict LFU
      //   evict = freqBuckets.get(minFreq).iterator().next()
      //   freqBuckets.get(minFreq).remove(evict); vals.remove(evict); freqs.remove(evict)
      // add new: vals.put(key, value); freqs.put(key, 1)
      //   freqBuckets.computeIfAbsent(1, k -> new LinkedHashSet<>()).add(key)
      //   minFreq = 1

Private helpers:
  - void incrementFreq(int key)
      // oldFreq = freqs.get(key); newFreq = oldFreq + 1
      // freqs.put(key, newFreq)
      // freqBuckets.get(oldFreq).remove(key)
      // if freqBuckets.get(oldFreq).isEmpty() && oldFreq == minFreq: minFreq++
      // freqBuckets.computeIfAbsent(newFreq, k -> new LinkedHashSet<>()).add(key)
```

**Design pattern:** LinkedHashSet per frequency bucket — O(1) eviction because `iterator().next()` returns the oldest insertion at that frequency level; combining two HashMaps achieves O(1) lookup and update.

**Trade-off:** LFU is harder to implement than LRU because it must track both frequency and recency. LRU only needs a doubly linked list ordered by recency; LFU needs an additional layer of bucketing by frequency with LRU tie-breaking within each bucket.

**Interview talking point:** "If asked why LFU is harder to implement than LRU, answer: LRU only needs to track recency — a doubly linked list does that in O(1). LFU must track both frequency and recency within the same frequency bucket. The key insight is using a LinkedHashSet per frequency level — insertion order within the set gives you the LRU ordering among ties, so you always evict the least-recently-used item at the minimum frequency in O(1)."

## Flashcards

| Q | A |
|---|---|
| What data structure enables O(1) LRU eviction? | A doubly linked list with sentinel head/tail; the node just before the tail sentinel is always the LRU item and can be removed in O(1). |
| Why use `LinkedHashSet` instead of `HashSet` in LFU frequency buckets? | `LinkedHashSet` preserves insertion order, making the first element the oldest at that frequency — enabling O(1) LRU tie-breaking without an extra linked list. |
| In "minimum adjacent swaps for K consecutive ones," why collect positions into array P instead of working on the raw array? | It reduces the problem from O(N) array swaps to O(K) median-cost on positions, and prefix sums make each window cost O(1). |
| What is the offset term in LC 1703 and why must it be subtracted? | The positions P[l..r] have gaps between them; adjacent swaps cannot skip indices. The offset `P[l]*K + K*(K-1)/2` accounts for the non-swap positional spread, leaving only the true swap count. |
| In the sliding-window frequency diff approach (LC 567), what does `nonZero` count and when is the answer found? | `nonZero` counts characters whose frequency differs between the current window and s1. The answer is found when `nonZero == 0` (all 26 character frequencies match). |
