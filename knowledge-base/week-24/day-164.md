## DSA (1.5 hours — OA Contest Mode)

### Pattern: Prefix/Suffix DP with Heap-Maintained Running Optimum
- Core idea: Precompute a prefix array where prefix[i] = optimal value (min/max sum of n elements) considering only nums[0..i], and a suffix array where suffix[i] = optimal value considering only nums[i..end]. Then sweep a split point and combine prefix[split] with suffix[split+1] to find the global optimum. Maintain the running optimum with a heap that evicts elements that should not be in the top-n set.
- Trigger condition: Use prefix/suffix DP with heap when the problem requires splitting an array at every possible split point and independently optimizing each half, where the optimization for each half can be maintained incrementally by tracking a sliding window of the best n elements.
- Time complexity: O(N log N) | Space complexity: O(N)

### Contest Template (write from memory before solving):
```java
// Prefix min-sum of n elements using a max-heap (evict the largest to keep n smallest)
int n = nums.length / 3;
long[] prefix = new long[nums.length];  // prefix[i] = min sum of n elements from nums[0..i]
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());
long runningSum = 0;
for (int i = 0; i < nums.length; i++) {
    maxHeap.offer(nums[i]);
    runningSum += nums[i];
    if (maxHeap.size() > n) runningSum -= maxHeap.poll(); // evict the largest
    if (maxHeap.size() == n) prefix[i] = runningSum;
}

// Suffix max-sum of n elements using a min-heap (evict the smallest to keep n largest)
long[] suffix = new long[nums.length];  // suffix[i] = max sum of n elements from nums[i..end]
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
runningSum = 0;
for (int i = nums.length - 1; i >= 0; i--) {
    minHeap.offer(nums[i]);
    runningSum += nums[i];
    if (minHeap.size() > n) runningSum -= minHeap.poll(); // evict the smallest
    if (minHeap.size() == n) suffix[i] = runningSum;
}

// Combine: split point i means left = nums[0..i], right = nums[i+1..end]
// Valid split points: i in [n-1, 2n-1] (left half has >= n elements, right half has >= n elements)
long ans = Long.MAX_VALUE;
for (int i = n - 1; i <= 2 * n - 1; i++) {
    ans = Math.min(ans, prefix[i] - suffix[i + 1]);
}
```

| 1 | Minimum Difference in Sums After Removal of Elements | 2163 | Hard (Cold Solve) | Prefix/Suffix Heap DP | n = len/3; prefix[i] = min sum of n elements from nums[0..i] using max-heap of size n; suffix[i] = max sum of n elements from nums[i..] using min-heap of size n; answer = min(prefix[i] - suffix[i+1]) for i in [n-1, 2n-1] |
| 2 | Partition Equal Subset Sum | 416 | Medium (Revision) | 1D Knapsack DP | REVISION: If total sum is odd return false; dp[j] = true if a subset sums to j; outer loop over each num, inner loop from target down to num (reverse to prevent reuse); answer = dp[target] |
| 3 | Longest Arithmetic Subsequence | 1027 | Medium (OA Timed) | 2D DP with HashMap per Index | OA: Set 20-min timer. dp[i] is a HashMap<Integer, Integer> mapping difference → longest arithmetic subsequence ending at i with that difference; for each j < i: diff = nums[i]-nums[j]; dp[i][diff] = dp[j].getOrDefault(diff, 1) + 1; answer = max value across all dp[i] entries + 1 is implicit in the stored values |

    //      Right half = nums[i+1..end], we pick the largest n from it → suffix[i+1] = max n-sum.
    //      Valid i: left half must have >= n elements (i >= n-1),
    //               right half must have >= n elements (len-1-(i+1)+1 >= n → i <= 2n-1).
    //      Use max-heap for prefix (evict max to keep n smallest).
    //      Use min-heap for suffix (evict min to keep n largest).

    // LC 416 — Partition Equal Subset Sum (revision — write from memory)
    public boolean canPartition(int[] nums) {
        int total = 0;
        for (int n : nums) total += n;
        if (total % 2 != 0) return false;
        int target = total / 2;
        boolean[] dp = new boolean[target + 1];
        dp[0] = true;
        for (int num : nums)
            for (int j = target; j >= num; j--)
                dp[j] = dp[j] || dp[j - num];
        return dp[target];
    }

#### Availability Query
```sql
-- Find all available rooms of a given type for given dates
SELECT r.room_id, r.room_number, r.floor
FROM Rooms r
WHERE r.room_type_id = :roomTypeId
  AND r.status = 'AVAILABLE'
  AND r.room_id NOT IN (
      SELECT res.room_id
      FROM Reservations res
      WHERE res.status NOT IN ('CANCELLED')
        AND res.check_in  < :checkOut   -- existing reservation starts before our checkout
        AND res.check_out > :checkIn    -- existing reservation ends after our checkin
  );
```

#### Overbooking Prevention
Two strategies — choose based on expected contention:

1. **Database-level:** Run availability check and INSERT in the same transaction with `SELECT ... FOR UPDATE` on the Rooms row. The row lock prevents concurrent transactions from booking the same room for overlapping dates.

2. **Redis distributed lock (high-demand scenarios):** Before writing to the DB, acquire a Redis lock:
   ```
   SET room:{hotel_id}:{room_type_id}:{check_in} LOCKED NX EX 30
   ```
   Only the request that gets the lock proceeds to the DB write. Others retry or receive "sold out". The TTL ensures the lock is always released even if the process crashes.

**Interview talking point:** "If asked how to handle high-demand bookings for a popular hotel where thousands try to reserve the last room simultaneously, answer: use a Redis distributed lock with TTL — SET room:{hotel}:{type}:{date} LOCKED NX EX 30 — only one request gets the lock, completes the DB write, then releases; others retry or receive 'sold out'. This prevents the thundering-herd problem where all concurrent requests hit the database simultaneously and race on the same row lock."

## Flashcards

| Q | A |
|---|---|
| In LC 2163, why do you use a max-heap for prefix and a min-heap for suffix? | Prefix = min sum of n elements → keep the n smallest → evict the largest when size exceeds n, so use a max-heap (the root is the largest, easy to evict). Suffix = max sum of n elements → keep the n largest → evict the smallest when size exceeds n, so use a min-heap (the root is the smallest, easy to evict). |
| What is the valid split point range in LC 2163 and why? | i ranges from n-1 to 2n-1 (inclusive). At i=n-1: left half is nums[0..n-1] (exactly n elements, the minimum needed). At i=2n-1: right half is nums[2n..3n-1] (exactly n elements, the minimum needed). Outside this range, one half has fewer than n elements and cannot contribute a valid n-element selection. |
| In LC 416 (Partition Equal Subset Sum), why must the inner loop run from target down to num? | The inner loop processes the current num only once per knapsack iteration (0/1 knapsack — each element used at most once). Iterating from high to low ensures that when dp[j] is updated using dp[j-num], dp[j-num] still reflects the state before the current num was considered. Iterating low to high would allow num to be used multiple times (unbounded knapsack). |
| In LC 1027 (Longest Arithmetic Subsequence), what does dp[i][diff] store? | The length of the longest arithmetic subsequence ending at index i with common difference diff. The value includes both endpoints, so a two-element subsequence [nums[j], nums[i]] stores 2. When computing dp[i][diff]: dp[j].getOrDefault(diff, 1) + 1 — the 1 default represents nums[j] alone (length 1 before appending nums[i]). |
| What two Redis lock properties make SET NX EX safe for hotel booking concurrency? | NX (Not eXists): only sets the key if it does not already exist — ensures mutual exclusion, only one request acquires the lock. EX (EXpiry in seconds): automatically releases the lock after the TTL even if the lock holder crashes — prevents deadlock where a crashed process holds the lock forever. |
