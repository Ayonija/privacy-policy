## DSA (1.5 hours — OA Contest Mode)

### Pattern: Sweep Line + Binary Search on Event Arrays
- Core idea: Decompose an interval problem into two sorted event arrays — one for interval starts and one for interval ends. To answer a point query at time t, use binary search to count how many intervals have started by t (upperBound on starts) and how many have already ended before t (lowerBound on ends). The active count at t is the difference: started - ended.
- Trigger condition: Use sweep line when the problem asks "how many intervals are active at a given point?" or involves counting overlapping ranges for a set of query points without needing the full interval list at query time.
- Time complexity: O((N + Q) log N) where N = number of intervals, Q = number of queries | Space complexity: O(N + Q)

### Contest Template (write from memory before solving):
```java
// Sweep line: count active intervals at each query point
int[] starts = /* extract and sort all interval start times */;
int[] ends   = /* extract and sort all interval end times */;

// For a query point t:
// bloomed = number of intervals whose start <= t  → upperBound(starts, t)
// done    = number of intervals whose end   <  t  → lowerBound(ends, t)
// active  = bloomed - done

// upperBound: first index where starts[i] > t  (i.e., count of starts <= t)
private int upperBound(int[] arr, int t) {
    int lo = 0, hi = arr.length;
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (arr[mid] <= t) lo = mid + 1;
        else hi = mid;
    }
    return lo; // number of elements <= t
}

// lowerBound: first index where ends[i] >= t  (i.e., count of ends < t)
private int lowerBound(int[] arr, int t) {
    int lo = 0, hi = arr.length;
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (arr[mid] < t) lo = mid + 1;
        else hi = mid;
    }
    return lo; // number of elements < t
}
```

    // LC 2251 — Number of Flowers in Full Bloom
    // (cold solve — 30-min timer, narrate before coding)

    // Key: Extract flowers[i][0] into starts[], flowers[i][1] into ends[]; sort both.
    //      For each person's time t: active = upperBound(starts, t) - lowerBound(ends, t).
    //      Person queries can be answered independently in any order — sort persons by time
    //      with their original index to produce the answer array in the right order.

    // LC 994 — Rotting Oranges (revision — write from memory)
    public int orangesRotting(int[][] grid) {
        int rows = grid.length, cols = grid[0].length;
        Queue<int[]> queue = new LinkedList<>();
        int fresh = 0;
        for (int r = 0; r < rows; r++)
            for (int c = 0; c < cols; c++) {
                if (grid[r][c] == 2) queue.offer(new int[]{r, c});
                else if (grid[r][c] == 1) fresh++;
            }
        int time = 0;
        int[][] dirs = {{1,0},{-1,0},{0,1},{0,-1}};
        while (!queue.isEmpty() && fresh > 0) {
            int size = queue.size();
            for (int i = 0; i < size; i++) {
                int[] cell = queue.poll();
                for (int[] d : dirs) {
                    int nr = cell[0] + d[0], nc = cell[1] + d[1];
                    if (nr >= 0 && nr < rows && nc >= 0 && nc < cols && grid[nr][nc] == 1) {
                        grid[nr][nc] = 2;
                        fresh--;
                        queue.offer(new int[]{nr, nc});
                    }
                }
            }
            time++;
        }
        return fresh == 0 ? time : -1;
    }

## System Design (30 min — LLD Deep Dive)

### Topic: Library Management System ER Diagram

#### Schema

```sql
-- Books
Books (
    isbn          VARCHAR(13)  PRIMARY KEY,
    title         VARCHAR(255) NOT NULL,
    author        VARCHAR(255) NOT NULL,
    genre         VARCHAR(100),
    total_copies  INT          NOT NULL DEFAULT 1,
    available_copies INT       NOT NULL DEFAULT 1
)

-- Patrons
Patrons (
    patron_id        BIGINT       PRIMARY KEY AUTO_INCREMENT,
    name             VARCHAR(255) NOT NULL,
    email            VARCHAR(255) UNIQUE NOT NULL,
    membership_type  ENUM('BASIC', 'PREMIUM') NOT NULL DEFAULT 'BASIC',
    max_books_allowed INT         NOT NULL DEFAULT 3,
    is_active        BOOLEAN      NOT NULL DEFAULT TRUE
)

-- Loans
Loans (
    loan_id       BIGINT       PRIMARY KEY AUTO_INCREMENT,
    patron_id     BIGINT       NOT NULL REFERENCES Patrons(patron_id),
    isbn          VARCHAR(13)  NOT NULL REFERENCES Books(isbn),
    checkout_date DATE         NOT NULL,
    due_date      DATE         NOT NULL,
    return_date   DATE,                          -- NULL if still on loan
    fine_amount   DECIMAL(8,2) NOT NULL DEFAULT 0.00
)

-- Reservations
Reservations (
    reservation_id BIGINT      PRIMARY KEY AUTO_INCREMENT,
    patron_id      BIGINT      NOT NULL REFERENCES Patrons(patron_id),
    isbn           VARCHAR(13) NOT NULL REFERENCES Books(isbn),
    reserved_at    TIMESTAMP   NOT NULL DEFAULT CURRENT_TIMESTAMP,
    status         ENUM('PENDING', 'FULFILLED', 'EXPIRED') NOT NULL DEFAULT 'PENDING'
)
```

#### Indexes
```sql
-- Fast lookups by book
CREATE INDEX idx_books_isbn ON Books(isbn);

-- Fast lookup of all loans for a patron (e.g., "how many books does this patron have?")
CREATE INDEX idx_loans_patron ON Loans(patron_id);

-- Fast lookup of active (unreturned) loans for a given book
CREATE INDEX idx_loans_active ON Loans(isbn, return_date)
    WHERE return_date IS NULL;  -- partial index: only indexes active loans

-- Reservation queue order per book
CREATE INDEX idx_reservations_isbn_time ON Reservations(isbn, reserved_at)
    WHERE status = 'PENDING';
```

#### Business Logic
- **Fine calculation:** `fine_amount = fine_per_day × MAX(0, DATEDIFF(return_date, due_date))`. If `return_date IS NULL` and today > due_date, compute fine against today for display but do not write it until return.
- **Reservation queue:** FIFO per isbn — ordered by `reserved_at ASC`. When a copy is returned (`available_copies` increments), query the first `PENDING` reservation for that isbn and notify the patron (email/SMS). If patron does not check out within 48 hours, set status to `EXPIRED` and notify the next in queue.
- **Concurrent checkout prevention:** Within a transaction — `SELECT available_copies FROM Books WHERE isbn=? FOR UPDATE` — if `available_copies > 0`, decrement and insert Loan record; else return "unavailable". The `FOR UPDATE` row-level lock ensures no two concurrent transactions can both read `available_copies > 0` for the same isbn.

**Interview talking point:** "If asked how to prevent two patrons checking out the last copy simultaneously, answer: use a database transaction with SELECT available_copies FROM Books WHERE isbn=? FOR UPDATE — this row-level lock ensures atomicity; one transaction decrements available_copies to 0 and commits; the other waits, reads 0, and returns 'no copies available'."
