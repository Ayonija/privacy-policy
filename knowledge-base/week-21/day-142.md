## DSA (1.5 hours)

### Pattern: 2D String DP — Pattern Matching
- Core idea: `dp[i][j]` = true if the first `i` characters of string `s` match the first `j` characters of pattern `p`. Transitions differ based on whether `p[j]` is a literal, `.`, or `*`. For `*`, you have two choices: use zero occurrences (look at `dp[i][j-2]`) or use one more occurrence (look at `dp[i-1][j]`).
- Trigger condition: String matches a regex or wildcard pattern, where `*` can expand to multiple characters. Classic tell: the problem says "match the entire input string."
- Time complexity: O(m×n) | Space complexity: O(m×n), row-optimizable to O(n)

### Pattern: Edit Distance (2D String DP — Transformation)
- Core idea: `dp[i][j]` = min operations to convert `s1[0..i-1]` to `s2[0..j-1]`. If `s1[i-1]==s2[j-1]`, copy for free; otherwise take min(insert, delete, replace) + 1.
- Trigger condition: "Minimum number of insertions/deletions/replacements to transform one string into another."
- Time complexity: O(m×n) | Space complexity: O(m×n), optimizable to O(min(m,n))

### Problems

| # | Problem | LC # | Difficulty | Pattern | Key Insight |
|---|---------|------|------------|---------|-------------|
| 1 | Regular Expression Matching | 10 | Hard (Cold Solve) | 2D String DP | `*` means zero-or-more of the PRECEDING element; handle `*` with two branches: skip pair (`dp[i][j-2]`) or match one more (`dp[i-1][j]` when `p[j-1]` matches `s[i-1]`) |
| 2 | Wildcard Matching | 44 | Hard (Cold Solve) | 2D String DP | `*` matches any sequence including empty; `dp[i][j] = dp[i-1][j] OR dp[i][j-1]` when `p[j-1]=='*'` |
| 3 | Edit Distance | 72 | Medium (Revision) | 2D String DP | REVISION: Three operations map to `dp[i-1][j]+1`, `dp[i][j-1]+1`, `dp[i-1][j-1]+1`; if chars match use `dp[i-1][j-1]` directly |

```java
    //   case '.': dp[i][j] = dp[i-1][j-1]
    //   case literal: dp[i][j] = dp[i-1][j-1] && s[i-1]==p[j-1]

    // LC 44 — Wildcard Matching (cold solve — delete this stub and rewrite from scratch)
    // TODO: boolean isMatch(String s, String p)
    // Key: dp[0][0]=true; dp[0][j]=dp[0][j-1] if p[j-1]=='*'
    //   case '*': dp[i][j] = dp[i-1][j] (use * for s[i-1]) OR dp[i][j-1] (use * for empty)
    //   case '?' or match: dp[i][j] = dp[i-1][j-1]

    // LC 72 — Edit Distance (revision — write from memory)
    public int minDistance(String word1, String word2) {
        int m = word1.length(), n = word2.length();
        int[][] dp = new int[m + 1][n + 1];
        for (int i = 0; i <= m; i++) dp[i][0] = i;
        for (int j = 0; j <= n; j++) dp[0][j] = j;
        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                if (word1.charAt(i-1) == word2.charAt(j-1))
                    dp[i][j] = dp[i-1][j-1];
                else
                    dp[i][j] = 1 + Math.min(dp[i-1][j-1], Math.min(dp[i-1][j], dp[i][j-1]));
            }
        }
        return dp[m][n];
    }
}
```

## System Design (1 hour)

### Topic: YouTube / Netflix — Video Ingestion & Transcoding Pipeline

#### Why Transcoding Is Hard
A user uploads a raw `.mov` file at 4K 60fps. Netflix/YouTube must serve that same video to a 2G phone user at 240p and a 4K OLED TV. That requires 15–20 different encoded files. Encoding is CPU-intensive: 1 hour of raw 4K video takes ~2–8 hours of CPU time to encode into all formats on a single machine.

#### Chunked Upload Flow

```
Client                    Upload Service              Object Storage (S3)
  │                            │                            │
  │──── Initiate Upload ───────►│                            │
  │◄─── Upload ID + Chunk URLs ─│                            │
  │                            │                            │
  │──── Chunk 1 (5MB) ─────────►│──── PUT chunk_1 ──────────►│
  │──── Chunk 2 (5MB) ─────────►│──── PUT chunk_2 ──────────►│
  │──── Chunk N (5MB) ─────────►│──── PUT chunk_N ──────────►│
  │                            │                            │
  │──── Complete Upload ────────►│                            │
  │                            │──── S3 Multipart Complete ─►│
  │                            │◄─── ETag / Version ID ──────│
  │                            │                            │
  │                            │──── Enqueue transcode job ─►│ (SQS/Kafka)
```

**Why chunks?** Resume on failure, parallel upload of multiple chunks, progress tracking. Typical chunk size: 5–10 MB.

**Interview talking point:** "If asked why encoding is done asynchronously, answer: A 1-hour raw video at 4K requires 2–8 CPU-hours of encoding. You cannot block the upload HTTP response for that long. Instead, the upload returns immediately after writing raw bytes to S3, then a Kafka message triggers a fleet of spot/preemptible transcoding workers that operate independently. Users see a 'Processing…' status until all renditions are ready."

## Flashcards

| Q | A |
|---|---|
| How do you handle `p[j-1] == '*'` in the regex matching DP (LC 10)? | Two branches: (1) zero occurrences: `dp[i][j] = dp[i][j-2]`; (2) one more occurrence: if `p[j-2]` matches `s[i-1]` (via `.` or equality), `dp[i][j] |= dp[i-1][j]`. |
| How does the wildcard `*` differ from regex `*` in the DP transition? | Wildcard `*` independently matches any sequence: `dp[i][j] = dp[i-1][j] OR dp[i][j-1]`. Regex `*` always refers to the preceding character/`.`. |
| What are the three operations in Edit Distance and which DP cell does each correspond to? | Insert → `dp[i][j-1]+1`; Delete → `dp[i-1][j]+1`; Replace → `dp[i-1][j-1]+1`. |
| Why does YouTube use chunked upload instead of a single PUT request? | Resume on failure, parallel chunk upload, progress tracking, and no single 4K raw file overwhelming one server connection. |
| What triggers transcoding in the YouTube/Netflix pipeline? | Upload Service writes raw video to S3, then publishes a message (Kafka/SQS) with video_id + S3 path. Transcode Coordinator consumes it and fans out per-rendition worker tasks. |
