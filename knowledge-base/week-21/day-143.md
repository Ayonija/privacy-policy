## DSA (1.5 hours)

### Pattern: Multi-Agent Grid DP
- Core idea: Instead of tracking two separate paths, parameterize both agents in the same DP state. For Cherry Pickup, use `dp[t][r1][r2]` where `t` is the step number and `r1`, `r2` are the row positions of agent 1 and agent 2 respectively. The column of each agent is derived as `c = t - r` (since each step moves right or down). This collapses a 4D problem to 3D.
- Trigger condition: Two entities traverse a grid simultaneously from (0,0) to (n-1,n-1); you want to maximize/minimize a combined objective without double-counting shared cells.
- Time complexity: O(n³) | Space complexity: O(n³), space-optimizable to O(n²)

### Pattern: Mathematical DP with Binary Search (Super Egg Drop)
- Core idea: Instead of `dp[k][n]` = min trials, invert to `dp[k][t]` = max floors you can check with `k` eggs and `t` trials. Recurrence: `dp[k][t] = dp[k-1][t-1] + dp[k][t-1] + 1`. Increase `t` until `dp[K][t] >= N`.
- Trigger condition: Combinatorial DP where the naive O(KN²) is too slow (N up to 10⁴). The inverted formulation is O(K log N).
- Time complexity: O(K log N) with binary search, O(K×N) naive | Space complexity: O(K)

    public int uniquePaths(int m, int n) {

        java.util.Arrays.fill(dp, 1);
        for (int i = 1; i < m; i++)

## System Design (1 hour)

### Topic: YouTube / Netflix — CDN & Adaptive Bitrate Streaming (ABR)

#### What Is Adaptive Bitrate Streaming?
ABR allows the player to switch video quality dynamically based on available bandwidth — no rebuffering when your network drops, no permanently low quality when your network recovers. The player downloads short segments (2–10 seconds) and can switch quality between segments.

#### HLS vs MPEG-DASH

| Feature | HLS (HTTP Live Streaming) | MPEG-DASH |
|---------|--------------------------|-----------|
| Creator | Apple | MPEG standard |
| Segment format | MPEG-TS or fMP4 | fMP4 |
| Manifest | `.m3u8` | `.mpd` |
| Apple device support | Native | Via library |
| DRM | FairPlay (Apple), Widevine | Widevine, PlayReady |
| Industry use | YouTube, Netflix (iOS) | Netflix (Android/Web) |

#### Manifest File Structure (HLS example)
```
#EXTM3U
#EXT-X-VERSION:3
#EXT-X-STREAM-INF:BANDWIDTH=800000,RESOLUTION=640x360
360p/index.m3u8
#EXT-X-STREAM-INF:BANDWIDTH=3000000,RESOLUTION=1280x720
720p/index.m3u8
#EXT-X-STREAM-INF:BANDWIDTH=6000000,RESOLUTION=1920x1080
1080p/index.m3u8
```

Each sub-manifest (`360p/index.m3u8`) lists individual segment URLs:
```
#EXTM3U
#EXT-X-TARGETDURATION:6
#EXTINF:6.006,
seg_001.ts
#EXTINF:6.006,
seg_002.ts
...
```

#### Adaptive Bitrate Algorithm (Client-Side)

```
Every 30 seconds (or on each segment download):

  measured_bandwidth = segment_bytes / download_time_seconds

  if measured_bandwidth > 1.2 × current_rendition_bitrate:
      upgrade quality  (if available)
  elif measured_bandwidth < 0.8 × current_rendition_bitrate:
      downgrade quality (if available)
  
  Target: buffer at least 30 seconds ahead at current quality
  If buffer < 10 seconds: immediately downgrade to lowest quality
```

**Netflix Open Connect Appliances (OCA):**
- 17,000+ appliances embedded inside ISP data centers worldwide.
- Netflix prefills OCAs with predicted popular content every night (proactive cache warming).
- During peak hours, 95%+ of Netflix bytes come from OCAs, never hitting the public internet.
- Each OCA: up to 100 TB SSD storage, 100 Gbps uplink to ISP core network.

#### Cache Strategy
| Content Type | TTL | Strategy |
|-------------|-----|----------|
| Video segments (immutable) | 1 year | Aggressive caching; URL includes content hash |
| Manifest files | 5–30 seconds | Short TTL; must reflect latest segment list |
| Thumbnails | 7 days | Moderate TTL; rarely change |
| Video metadata | 60 seconds | Cached at app tier, not CDN |

#### Key Latency Targets
- Time to first byte from CDN edge: < 50ms
- Time to first frame (TTFF): < 2 seconds
- Rebuffer ratio: < 0.5% of playback time (Netflix target: < 0.1%)
- Segment download should complete in < 50% of segment duration (6s segment → < 3s download)

#### Streaming Request Flow
```
1. Client hits API → receives signed CDN URL for manifest
2. Client fetches manifest.m3u8 from CDN edge
3. Client picks initial quality (usually lowest or based on recent history)
4. Client fetches segment_001.ts from CDN edge
5. CDN edge: cache HIT → serve immediately; cache MISS → fetch from origin, cache, serve
6. Client measures download speed → ABR algorithm selects quality for next segment
7. Repeat from step 4
```

**Interview talking point:** "If asked how Netflix achieves < 2-second time-to-first-frame globally, answer: OCAs embedded inside ISPs eliminate network hops. Clients probe bandwidth before playback starts and select the highest quality that fits. Manifest files have 30-second TTL so the player always has current segment URLs. Netflix pre-warms OCAs with predicted popular content nightly so cache hit rates are > 95% during peak hours."
