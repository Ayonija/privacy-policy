## DSA (1.5 hours)

### Pattern: Interval DP (Split-Point Optimization)
- Core idea: `dp[i][j]` represents the optimal answer for the subarray/substring from index `i` to `j`. You iterate over all possible "last" or "split" points `k` within `[i,j]` and combine subproblem results.
- Trigger condition: The problem asks for an optimal cost/value over a contiguous range, and the order of operations matters (e.g., which balloon to burst last, which matrix to multiply last).
- Time complexity: O(n³) | Space complexity: O(n²)

### Pattern: Longest Common Subsequence (Classic DP)
- Core idea: `dp[i][j]` = LCS length of `s1[0..i-1]` and `s2[0..j-1]`. If characters match, `dp[i][j] = dp[i-1][j-1] + 1`; otherwise `max(dp[i-1][j], dp[i][j-1])`.
- Trigger condition: "Find the longest subsequence common to two strings/sequences." Also used as a building block for edit distance and diff tools.
- Time complexity: O(m×n) | Space complexity: O(m×n), optimizable to O(min(m,n))

### Problems

| # | Problem | LC # | Difficulty | Pattern | Key Insight |
|---|---------|------|------------|---------|-------------|
| 1 | Burst Balloons | 312 | Hard (Cold Solve) | Interval DP | Think in reverse: which balloon is burst LAST in range [i,j]; `dp[i][j] += nums[i-1]*nums[k]*nums[j+1]` |
| 2 | Strange Printer | 664 | Hard (Cold Solve) | Interval DP | If `s[i]==s[j]`, the last print covering `j` can also cover `i` for free: `dp[i][j] = dp[i][j-1]` |
| 3 | Longest Common Subsequence | 1143 | Medium (Revision) | 2D DP | REVISION: Build `dp[i][j]` bottom-up; final answer is `dp[m][n]` |

```java
    // Key: pad nums with 1 on both ends; dp[i][j] = max coins from bursting ALL balloons strictly inside (i,j)
    //      for each k in (i,j): dp[i][j] = max(dp[i][j], dp[i][k] + nums[i]*nums[k]*nums[j] + dp[k][j])

    // LC 664 — Strange Printer (cold solve — delete this stub and rewrite from scratch)
    // TODO: int strangePrinter(String s)
    // Key: dp[i][j] = min turns to print s[i..j]
    //      base: dp[i][i] = 1
    //      if s[i]==s[j]: dp[i][j] = dp[i][j-1]  (extend last print of i to cover j for free)
    //      else: dp[i][j] = min over k in [i,j-1] of dp[i][k] + dp[k+1][j]

    // LC 1143 — Longest Common Subsequence (revision — write from memory)
    public int longestCommonSubsequence(String text1, String text2) {
        int m = text1.length(), n = text2.length();
        int[][] dp = new int[m + 1][n + 1];
        for (int i = 1; i <= m; i++)
            for (int j = 1; j <= n; j++)
                dp[i][j] = text1.charAt(i-1) == text2.charAt(j-1)
                    ? dp[i-1][j-1] + 1
                    : Math.max(dp[i-1][j], dp[i][j-1]);
        return dp[m][n];
    }
}
```

## System Design (1 hour)

### Topic: YouTube / Netflix — Requirements, Scale Numbers & Core Architecture Overview

#### Functional Requirements
- Users can upload videos (YouTube) or studios deliver content (Netflix).
- Users can stream videos at adaptive quality.
- Users can search, browse, and get recommendations.
- Users can like, comment, and share (YouTube focus).
- System tracks watch history and resumes playback.

#### Non-Functional Requirements
- High availability (99.99% uptime for streaming).
- Low latency playback start (< 2 seconds to first frame).
- Eventual consistency acceptable for likes/views counts.
- Durability: uploaded video must never be lost.
- Global scale with geo-distributed delivery.

#### Scale Numbers (memorize these)

| Metric | YouTube | Netflix |
|--------|---------|---------|
| Users | 2B logged-in users | 250M subscribers |
| Upload rate | 500 hours of video per minute | Studio pipeline (not user upload) |
| Watch time | 1B hours watched per day | 15% of global internet traffic |
| Titles | 800M+ videos | 15,000+ titles |
| Peak streaming QPS | ~10M concurrent streams | ~15M concurrent streams |
| Video formats per upload | 15–20 (360p → 4K + HDR) | 1,200+ encoding profiles |

**Back-of-envelope:**
- 1B hours/day = ~42M hours/hour = ~700K hours/minute of video streamed
- 500 hours uploaded/minute → ~8 hours/second → each triggers 15-20 transcode jobs
- At 1 Mbps avg bitrate, 1B hours/day = 1B × 3600s × 1Mbps / 8 = ~450 PB/day egress

#### Service Decomposition
| Service | Responsibility |
|---------|---------------|
| Upload Service | Receives chunked video, writes to object storage, enqueues transcode job |
| Transcode Service | Converts raw video into 15-20 ABR renditions, writes back to storage |
| Streaming Service | Generates signed URLs, manages session state, reports progress |
| Metadata Service | Stores video title, description, tags, owner, view count |
| Recommendation Service | Collaborative filtering engine, feeds homepage |
| Search Service | Elasticsearch-backed full-text search with trending signals |
| CDN Layer | Caches encoded segments close to users; Netflix uses Open Connect (17,000+ appliances) |

**Interview talking point:** "If asked how YouTube achieves 1B hours/day of streaming, answer: 99% of bytes come from CDN edge caches — the origin only serves cache misses. Netflix's Open Connect Appliances (OCAs) are embedded directly inside ISP networks, so traffic never hits the public internet."

## Flashcards

| Q | A |
|---|---|
| How do you set up the Interval DP state for Burst Balloons (LC 312)? | Pad array with 1s on both ends. `dp[i][j]` = max coins bursting all balloons strictly between i and j. Iterate over k as the LAST balloon burst in that range. |
| How do you derive the Strange Printer recurrence when `s[i] == s[j]`? | `dp[i][j] = dp[i][j-1]` — the turn that prints position i can extend to cover j at no extra cost. |
| What is the time complexity of interval DP and why? | O(n³): n² states (all pairs i,j) × O(n) to try each split point k. |
| How do you compute LCS for two strings of lengths m and n? | `dp[i][j] = dp[i-1][j-1]+1` if chars match, else `max(dp[i-1][j], dp[i][j-1])`. Return `dp[m][n]`. |
| What YouTube scale number should you state first in a system design interview? | 500 hours of video uploaded per minute and 1B hours watched per day — these two numbers establish upload pipeline and streaming CDN requirements. |
