## DSA (2 hours)

### Pattern: Advanced LCS Variants — 3D State and Interval DP on Strings

**Core idea:**
Beyond standard LCS, some problems require a third dimension in the state:
- **Interleaving String:** merge two strings into a third — `dp[i][j]` suffices
- **Remove Boxes:** the "attached boxes" count changes how we score removal — need `dp[l][r][k]`
- **Max Dot Product:** LCS skeleton but with numeric products and a zero-extension guard

**Trigger condition:**
- "Can string s3 be formed by merging s1 and s2 in order?" → 2D interleaving DP
- "Remove groups of same-colored boxes for maximum points, grouping non-contiguous boxes" → 3D interval DP
- "Max dot product of non-empty common subsequences" → LCS-style with product accumulation

### Full Solution Walkthrough — Interleaving String (LC 97) — Revision

**Problem:** Given s1, s2, s3: can s3 be formed by interleaving s1 and s2 (maintaining relative order of each)?

**State:** `dp[i][j]` = can s3[0..i+j-1] be formed by interleaving s1[0..i-1] and s2[0..j-1].

**Recurrence:**
```
dp[i][j] = (dp[i-1][j] AND s1[i-1] == s3[i+j-1])   # use s1[i-1]
         OR (dp[i][j-1] AND s2[j-1] == s3[i+j-1])   # use s2[j-1]
```

```java
class Solution {
    public static boolean isInterleave(String s1, String s2, String s3) {
        int m = s1.length(), n = s2.length();
        if (m + n != s3.length()) return false;

        boolean[][] dp = new boolean[m + 1][n + 1];
        dp[0][0] = true;

        for (int i = 1; i <= m; i++)
            dp[i][0] = dp[i - 1][0] && s1.charAt(i - 1) == s3.charAt(i - 1);
        for (int j = 1; j <= n; j++)
            dp[0][j] = dp[0][j - 1] && s2.charAt(j - 1) == s3.charAt(j - 1);

        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                dp[i][j] = (dp[i - 1][j] && s1.charAt(i - 1) == s3.charAt(i + j - 1)) ||
                           (dp[i][j - 1] && s2.charAt(j - 1) == s3.charAt(i + j - 1));
            }
        }

        return dp[m][n];
    }
}
```

**1D space optimization:** Since dp[i][j] only depends on dp[i-1][j] and dp[i][j-1], can compress to a single row.

### Full Solution Walkthrough — Max Dot Product (LC 1458) — Hard

**Problem:** Find the maximum dot product of two non-empty subsequences of nums1 and nums2 (subsequences aligned element-wise).

**State:** `dp[i][j]` = max dot product using non-empty subsequences of nums1[0..i-1] and nums2[0..j-1].

**Recurrence:**
```
product = nums1[i-1] * nums2[j-1]
dp[i][j] = max(
    product,                           # use only these two elements
    max(0, dp[i-1][j-1]) + product,   # extend previous (only if positive)
    dp[i-1][j],                        # skip nums1[i-1]
    dp[i][j-1],                        # skip nums2[j-1]
)
```

**Key insight:** `max(0, dp[i-1][j-1])` — if the previous best dot product was negative, starting fresh (just this product) is better than extending.

```java
class Solution {
    public static int maxDotProduct(int[] nums1, int[] nums2) {
        int m = nums1.length, n = nums2.length;
        // Initialize to Integer.MIN_VALUE since subsequences must be non-empty
        int[][] dp = new int[m + 1][n + 1];
        for (int[] row : dp)
            java.util.Arrays.fill(row, Integer.MIN_VALUE);

        for (int i = 1; i <= m; i++) {
            for (int j = 1; j <= n; j++) {
                int product = nums1[i - 1] * nums2[j - 1];
                dp[i][j] = Math.max(product,
                           Math.max(Math.max(0, dp[i - 1][j - 1]) + product,
                           Math.max(dp[i - 1][j], dp[i][j - 1])));
            }
        }

        return dp[m][n];
    }
}
```

**Edge case:** If all products are negative, we must still return one — the max single product. The `dp[i-1][j]` and `dp[i][j-1]` transitions propagate previous bests, ensuring the max negative is accessible.

### Full Solution Walkthrough — Remove Boxes (LC 546) — Hard

**Problem:** Given boxes with colors (integers), remove boxes in groups of same color to score k² points for a group of size k. Maximize total score.

**Why naive interval DP fails:** Removing `boxes[l..r]` optimally doesn't depend only on l and r — it depends on how many boxes outside [l..r] are the same color as boxes[l] and can be grouped with it.

**State:** `dp[l][r][k]` = max points from `boxes[l..r]` given that k boxes with the same color as `boxes[l]` are **already attached to the left** of `boxes[l]`.

**Recurrence:**
```
First: absorb consecutive same-color boxes at start (they always join boxes[l]'s group):
    while l < r and boxes[l+1] == boxes[l]: l++, k++

Base case: l > r → 0

Option 1: Remove boxes[l] with its k attached boxes = (k+1)² + dp[l+1][r][0]

Option 2: For each m in [l+1, r] where boxes[m] == boxes[l]:
    # Move middle segment boxes[l+1..m-1] out of the way, then attach boxes[m] to boxes[l]
    candidate = dp[l+1][m-1][0] + dp[m][r][k+1]
```

```java
class Solution {
    int[] boxes;
    int[][][] memo;

    public int removeBoxes(int[] boxes) {
        this.boxes = boxes;
        int n = boxes.length;
        memo = new int[n][n][n];
        return dp(0, n - 1, 0);
    }

    private int dp(int l, int r, int k) {
        if (l > r) return 0;
        if (memo[l][r][k] != 0) return memo[l][r][k];

        // Absorb consecutive same-color boxes at start
        while (l < r && boxes[l + 1] == boxes[l]) {
            l++;
            k++;
        }
        // Option 1: remove boxes[l] group immediately
        int result = (k + 1) * (k + 1) + dp(l + 1, r, 0);
        // Option 2: find any m with boxes[m] == boxes[l] and merge
        for (int m = l + 1; m <= r; m++) {
            if (boxes[m] == boxes[l]) {
                result = Math.max(result, dp(l + 1, m - 1, 0) + dp(m, r, k + 1));
            }
        }
        memo[l][r][k] = result;
        return result;
    }
}
```

**Intuition for Option 2:** By finding another box of the same color as boxes[l], we can "bridge" over the middle segment. We compute the cost to clear boxes[l+1..m-1] independently (dp[l+1][m-1][0]), and then treat boxes[m..r] with k+1 leading same-color boxes (since boxes[l]'s group will join boxes[m]).

**Complexity:** O(n⁴) states × O(n) transitions = O(n⁵) in the worst case, but the early-absorption optimization brings it to ~O(n⁴) in practice.

## System Design (30 min)

### Topic: API Gateway Design — Auth, Routing, Throttling

**What is an API Gateway?**
A single entry point that sits between clients and backend services. Handles cross-cutting concerns so individual services don't have to.

**Core responsibilities:**
1. **Authentication/Authorization:** Validate JWT or API key before forwarding. Return 401/403 without hitting backend.
2. **Routing:** Map `/users` → User Service, `/orders` → Order Service. Can do path rewriting.
3. **Rate limiting:** Enforce per-user or per-endpoint limits (see Day 104–105). Reject before hitting compute.
4. **Request/response transformation:** Add headers, strip internal fields, gzip, protocol translation (REST → gRPC).
5. **Load balancing:** Round-robin, least-connections, or weighted across backend instances.
6. **Observability:** Log all requests centrally; emit latency histograms; propagate trace IDs.

**Architecture:**
```
Client → API Gateway → [Auth Service]
                     → [Rate Limiter (Redis)]
                     → Backend Service A
                     → Backend Service B
                     → Backend Service C
```

**Trade-offs vs. service mesh:**
| | API Gateway | Service Mesh (Istio/Envoy) |
|--|------------|--------------------------|
| Position | North-south (client→service) | East-west (service→service) |
| Manages | External API contract | Internal service communication |
| Complexity | Lower | Higher (sidecar proxies) |
| Use case | API versioning, external auth | mTLS, internal retries |

**Interview talking point:**
*"The API Gateway is where you enforce the contract between external clients and internal services. Never let rate limiting or auth logic bleed into individual services — centralize it. At scale, the gateway itself becomes a bottleneck, so it must be stateless (all state in Redis/external) and horizontally scalable. Use a CDN in front of it to absorb static traffic before it hits the gateway."*
