### Code Skeleton

```java
class Solution {

    // LC 41 — First Missing Positive (cold solve — delete this stub and rewrite from scratch)
    // TODO: int firstMissingPositive(int[] nums)
    // Key: treat array as hash; nums[nums[i]-1] = nums[i]
    //   Phase 1: for i in [0..n-1]: while nums[i] is in [1..n] and nums[nums[i]-1] != nums[i]: swap(nums, i, nums[i]-1)
    //   Phase 2: for i in [0..n-1]: if nums[i] != i+1: return i+1
    //   Return n+1 (all positions 1..n are filled correctly)
    // Note: each element is swapped at most once → O(n) total
    public int firstMissingPositive(int[] nums) {
        int n = nums.length;
        for (int i = 0; i < n; i++) {
            while (nums[i] > 0 && nums[i] <= n && nums[nums[i] - 1] != nums[i]) {
                int tmp = nums[nums[i] - 1];
                nums[nums[i] - 1] = nums[i];
                nums[i] = tmp;
            }
        }
        for (int i = 0; i < n; i++) {
            if (nums[i] != i + 1) return i + 1;
        }
        return n + 1;
    }

    // LC 15 — 3Sum (revision — write from memory)
    public List<List<Integer>> threeSum(int[] nums) {
        Arrays.sort(nums);
        List<List<Integer>> res = new ArrayList<>();
        for (int i = 0; i < nums.length - 2; i++) {
            if (i > 0 && nums[i] == nums[i-1]) continue;
            int left = i+1, right = nums.length-1;
            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];
                if (sum == 0) {
                    res.add(Arrays.asList(nums[i], nums[left], nums[right]));
                    while (left < right && nums[left] == nums[left+1]) left++;
                    while (left < right && nums[right] == nums[right-1]) right--;
                    left++; right--;
                } else if (sum < 0) left++;
                else right--;
            }
        }
        return res;
    }

    // LC 238 — Product of Array Except Self (OA timed — set 20-minute timer)
    // TODO: int[] productExceptSelf(int[] nums)
    // Key: output[i] = left product of all elements before i × right product of all elements after i
    //   Pass 1: output[0] = 1; for i in [1..n-1]: output[i] = output[i-1] * nums[i-1]
    //   Pass 2: right = 1; for i from n-1 down to 0: output[i] *= right; right *= nums[i]
    public int[] productExceptSelf(int[] nums) {
        int n = nums.length;
        int[] output = new int[n];
        output[0] = 1;
        for (int i = 1; i < n; i++) output[i] = output[i-1] * nums[i-1];
        int right = 1;
        for (int i = n - 1; i >= 0; i--) { output[i] *= right; right *= nums[i]; }
        return output;
    }
}
```

## System Design (30 min)

### Topic: Load Balancing & API Gateway — Component Deep-Dive

#### Load Balancer Layers

| Layer | Protocol | Visibility | Example Use |
|-------|----------|------------|-------------|
| L4 (Transport) | TCP/UDP | IP + port only | Fast; no HTTP awareness; cannot route by URL |
| L7 (Application) | HTTP/HTTPS | Headers, URL, cookies | Route by path (`/api/v1` vs `/api/v2`), cookie-based sticky sessions |

**L7 advantage:** Can route `/upload` to Upload Service, `/stream` to Streaming Service — impossible at L4.

#### Load Balancing Algorithms

| Algorithm | How it works | When to use |
|-----------|-------------|-------------|
| Round-robin | Requests distributed in order across backends | Stateless services with similar request cost |
| Least-connections | Route to backend with fewest active connections | Long-lived connections (WebSockets, file uploads) |
| IP hash | Hash client IP to select backend | Stateful sessions without external session store |
| Weighted round-robin | Distribute proportionally to backend capacity | Mixed instance types (some backends are larger) |

#### Sticky Sessions & Their Problem

- Sticky sessions via IP hash: same client always routes to same backend.
- **Problem in auto-scaling:** adding or removing a node rehashes client IPs → existing sessions route to the wrong node → session state lost.
- **Solution:** externalize session state to Redis. Any backend can serve any client because session data is in Redis, not server memory.

#### Health Checks

- **Passive:** detect failure when a connection attempt fails (reactive; some requests fail before detection).
- **Active:** load balancer sends periodic pings (`GET /health`) to each backend; remove from pool if N consecutive failures (proactive).

#### API Gateway

Single entry point for all client traffic. Responsibilities:

```
Client Request
     │
     ▼
┌────────────────────────────────┐
│           API GATEWAY          │
│  1. TLS/SSL Termination        │  ← decrypt once here; backends use plain HTTP
│  2. Authentication (JWT verify)│  ← validate token before reaching services
│  3. Rate Limiting              │  ← token bucket per user/IP; block abuse
│  4. Request Routing            │  ← /upload → Upload Svc; /stream → Stream Svc
│  5. Request/Response Transform │  ← header injection, response shaping
│  6. Observability              │  ← log, trace, metrics per request
└────────────────────────────────┘
     │
     ├──► Upload Service
     ├──► Streaming Service
     └──► Metadata Service
```

**Interview talking point:** "If asked why sticky sessions break in auto-scaling, answer: adding/removing nodes rehashes client IPs, routing existing sessions to wrong nodes. Solution: externalize session state to Redis so any backend can serve any client."

## Flashcards

| Q | A |
|---|---|
| What is the key invariant that makes LC 41 First Missing Positive O(n) despite the nested while loop? | Each element is swapped into its correct position at most once. Once `nums[i] == i+1`, that element is never moved again. So total swaps across all iterations of the outer loop is at most n. |
| In 3Sum, after finding a zero-sum triplet, how do you skip duplicates before moving left and right pointers? | Skip while `nums[left] == nums[left+1]` (increment left) and while `nums[right] == nums[right-1]` (decrement right), then do the final `left++; right--`. |
| How does LC 238 Product of Array Except Self achieve O(1) extra space? | Use the output array as the left-pass accumulator: `output[i] = product of all nums[0..i-1]`. Then do a right pass with a single `right` variable: `output[i] *= right; right *= nums[i]`. No extra arrays needed. |
| What is the difference between L4 and L7 load balancers? | L4 operates at TCP/UDP (sees IP + port only, cannot inspect HTTP). L7 operates at HTTP (sees headers, URL, cookies; can route by path or cookie; enables sticky sessions by cookie). |
| Why do sticky sessions via IP hash break when auto-scaling adds a new backend node? | Adding a node changes the hash ring mapping; the same client IP now hashes to a different backend that does not have the session. Fix: externalize session state to Redis so any node can serve any client. |
