### Pattern Revision: BFS Shortest Path + Union-Find + Kahn's Topological Sort

**Union-Find template (reproduce from memory before reading the problem):**

```java
// Union-Find template
int[] parent, rank;
void init(int n) {
    parent = new int[n];
    rank = new int[n];
    for (int i = 0; i < n; i++) parent[i] = i;
}
int find(int x) {
    return parent[x] == x ? x : (parent[x] = find(parent[x])); // path compression
}
void union(int x, int y) {
    int px = find(x), py = find(y);
    if (px == py) return;
    if (rank[px] < rank[py]) { int t = px; px = py; py = t; } // union by rank
    parent[py] = px;
    if (rank[px] == rank[py]) rank[px]++;
}
```

**Kahn's BFS topological sort template (reproduce from memory before reading the problem):**

```java
// Kahn's BFS topological sort template
int[] inDegree = new int[n]; // build graph + fill inDegree
Queue<Integer> q = new LinkedList<>();
for (int i = 0; i < n; i++) if (inDegree[i] == 0) q.offer(i);
List<Integer> order = new ArrayList<>();
while (!q.isEmpty()) {
    int u = q.poll();
    order.add(u);
    for (int v : graph.get(u)) {
        if (--inDegree[v] == 0) q.offer(v);
    }
}
// cycle detected if order.size() != n
```

### Code Skeleton

```java
class Solution {

    // LC 2045 — Second Minimum Time to Reach Destination (cold solve — delete this stub and rewrite from scratch)
    // TODO: int secondMinimum(int n, int[][] edges, int time, int change)
    // Key:
    //   Build adjacency list (1-indexed, undirected)
    //   dist1[i] = first minimum time to reach node i; dist2[i] = second minimum time
    //   BFS: process (node, currentTime); for each neighbor:
    //     Compute actual arrival time accounting for signal:
    //       int period = currentTime / change;
    //       if (period % 2 == 1): must wait for green → waitTime = (period + 1) * change
    //       else: waitTime = currentTime (already green)
    //       newTime = waitTime + time
    //     If newTime < dist1[neighbor]: dist2[neighbor] = dist1[neighbor]; dist1[neighbor] = newTime; enqueue
    //     Else if newTime < dist2[neighbor] && newTime != dist1[neighbor]: dist2[neighbor] = newTime; enqueue
    //   Return dist2[n]
    public int secondMinimum(int n, int[][] edges, int time, int change) {
        List<List<Integer>> graph = new ArrayList<>();
        for (int i = 0; i <= n; i++) graph.add(new ArrayList<>());
        for (int[] e : edges) { graph.get(e[0]).add(e[1]); graph.get(e[1]).add(e[0]); }
        int[] dist1 = new int[n + 1], dist2 = new int[n + 1];
        Arrays.fill(dist1, Integer.MAX_VALUE); Arrays.fill(dist2, Integer.MAX_VALUE);
        dist1[1] = 0;
        Queue<int[]> q = new LinkedList<>();
        q.offer(new int[]{1, 0});
        while (!q.isEmpty()) {
            int[] curr = q.poll();
            int node = curr[0], t = curr[1];
            // Compute next green light time
            int period = t / change;
            int waitTime = (period % 2 == 1) ? (period + 1) * change : t;
            int newTime = waitTime + time;
            for (int neighbor : graph.get(node)) {
                if (newTime < dist1[neighbor]) {
                    dist2[neighbor] = dist1[neighbor];
                    dist1[neighbor] = newTime;
                    q.offer(new int[]{neighbor, newTime});
                } else if (newTime < dist2[neighbor] && newTime != dist1[neighbor]) {
                    dist2[neighbor] = newTime;
                    q.offer(new int[]{neighbor, newTime});
                }
            }
        }
        return dist2[n];
    }

    // LC 200 — Number of Islands (revision — write from memory)
    public int numIslands(char[][] grid) {
        int m = grid.length, n = grid[0].length, count = 0;
        for (int r = 0; r < m; r++) {
            for (int c = 0; c < n; c++) {
                if (grid[r][c] == '1') {
                    count++;
                    bfs(grid, r, c);
                }
            }
        }
        return count;
    }
    private void bfs(char[][] grid, int r, int c) {
        int m = grid.length, n = grid[0].length;
        Queue<int[]> q = new LinkedList<>();
        q.offer(new int[]{r, c});
        grid[r][c] = '0';
        int[][] dirs = {{1,0},{-1,0},{0,1},{0,-1}};
        while (!q.isEmpty()) {
            int[] cell = q.poll();
            for (int[] d : dirs) {
                int nr = cell[0] + d[0], nc = cell[1] + d[1];
                if (nr >= 0 && nr < m && nc >= 0 && nc < n && grid[nr][nc] == '1') {
                    grid[nr][nc] = '0';
                    q.offer(new int[]{nr, nc});
                }
            }
        }
    }

    // LC 417 — Pacific Atlantic Water Flow (OA timed — set 20-minute timer)
    // TODO: List<List<Integer>> pacificAtlantic(int[][] heights)
    // Key: reverse BFS — start from ocean borders, flow uphill (height >= current cell)
    //   Pacific: top row + left column. Atlantic: bottom row + right column.
    //   BFS from each set; mark reachable cells.
    //   Result: cells reachable from both.
    public List<List<Integer>> pacificAtlantic(int[][] heights) {
        int m = heights.length, n = heights[0].length;
        boolean[][] pacific = new boolean[m][n], atlantic = new boolean[m][n];
        Queue<int[]> pq = new LinkedList<>(), aq = new LinkedList<>();
        for (int r = 0; r < m; r++) {
            pq.offer(new int[]{r, 0}); pacific[r][0] = true;
            aq.offer(new int[]{r, n-1}); atlantic[r][n-1] = true;
        }
        for (int c = 0; c < n; c++) {
            pq.offer(new int[]{0, c}); pacific[0][c] = true;
            aq.offer(new int[]{m-1, c}); atlantic[m-1][c] = true;
        }
        bfsOcean(heights, pq, pacific);
        bfsOcean(heights, aq, atlantic);
        List<List<Integer>> result = new ArrayList<>();
        for (int r = 0; r < m; r++)
            for (int c = 0; c < n; c++)
                if (pacific[r][c] && atlantic[r][c]) result.add(Arrays.asList(r, c));
        return result;
    }
    private void bfsOcean(int[][] h, Queue<int[]> q, boolean[][] visited) {
        int[][] dirs = {{1,0},{-1,0},{0,1},{0,-1}};
        int m = h.length, n = h[0].length;
        while (!q.isEmpty()) {
            int[] cell = q.poll();
            for (int[] d : dirs) {
                int nr = cell[0]+d[0], nc = cell[1]+d[1];
                if (nr>=0 && nr<m && nc>=0 && nc<n && !visited[nr][nc] && h[nr][nc]>=h[cell[0]][cell[1]]) {
                    visited[nr][nc] = true;
                    q.offer(new int[]{nr, nc});
                }
            }
        }
    }
}
```

## System Design (30 min)

### Topic: Message Queues & Event-Driven Architecture (Kafka) — Component Deep-Dive

#### Kafka Core Concepts

```
Topic: logical stream of messages
  └─ Partition 1: [msg0, msg1, msg2, msg3, ...]   ← ordered log, append-only
  └─ Partition 2: [msg0, msg1, ...]
  └─ Partition 3: [msg0, msg1, ...]

Consumer Group A:                 Consumer Group B:
  Consumer 1 → Partition 1          Consumer X → Partition 1,2,3
  Consumer 2 → Partition 2          (reads all messages independently)
  Consumer 3 → Partition 3
```

**Key rule:** One partition is owned by exactly one consumer in a group at a time. Maximum parallelism = number of partitions.

#### Delivery Semantics

| Semantic | Mechanism | Use when |
|----------|-----------|----------|
| At-most-once | Commit offset before processing | Loss acceptable; no duplicates tolerated (e.g., analytics where missing 0.01% of events is fine) |
| At-least-once (default) | Commit offset after processing | Duplicates OK if processing is idempotent; crash before commit → redelivery |
| Exactly-once | Kafka transactions + idempotent producers | Duplicates cause business harm (payments, inventory deduction); higher latency cost |

**Idempotent processing for at-least-once:** Use a deduplication key (e.g., `message_id`) checked against a Redis set or DB unique constraint before processing.

#### Dead Letter Queue (DLQ)

```
Producer → Topic → Consumer attempts processing
                        ↓ fails
                   Retry (N times, exponential backoff)
                        ↓ still fails
                   Route to DLQ (Dead Letter Topic)
                        ↓
                   Monitoring alert → manual inspection
```

**Purpose:** Prevent a "poison-pill" message (one that always fails) from blocking all subsequent messages in the partition. Without DLQ, one bad message stalls the entire partition indefinitely.

#### Backpressure

```
Consumers lagging → consumer_lag metric spikes

Fix options (in order):
  1. Add more consumers to the consumer group (up to partition count)
  2. If consumers = partitions and still lagging → add partitions to topic
     (requires topic reconfiguration; rebalances partition ownership)
  3. Optimize consumer processing (batch DB writes, async I/O)
```

#### Ordering Guarantee

**Within a partition:** Strictly ordered (single append-only log consumed sequentially).
**Across partitions:** No ordering guarantee.

**Partition key design:** Route ordering-sensitive events to the same partition using a business key (e.g., `user_id` for user activity, `order_id` for order events). All events for the same key land on the same partition → ordered.

**Interview talking point:** "If asked why Kafka is ordered within a partition but not across partitions, answer: each partition is a single append-only log consumed sequentially; ordering across partitions would require global coordination, destroying horizontal scale. Design your partition key so ordering-sensitive events share a key (e.g., user_id)."

## Flashcards

| Q | A |
|---|---|
| In LC 2045 Second Minimum Time, how do you compute the wait time at a node when you arrive at time `t` and the signal change period is `change`? | `int period = t / change; int waitTime = (period % 2 == 1) ? (period + 1) * change : t;` — if you arrive during a red phase (odd period), wait until the next green; if already green, proceed immediately. |
| In LC 417 Pacific Atlantic Water Flow, why do you do BFS from the ocean borders instead of BFS from each cell toward the ocean? | Forward flow from each cell would require checking all possible paths downhill. Reverse flow from the ocean border is simpler: BFS uphill (to cells with height >= current) naturally finds all cells that can drain to that ocean. Intersection of the two reachable sets gives the answer. |
| What is the Union-Find path compression optimization and what does it achieve? | During `find(x)`, set `parent[x] = find(parent[x])` recursively, making every node on the path point directly to the root. This flattens the tree so future finds are O(1). Combined with union by rank, amortized cost per operation is O(α(n)) ≈ constant. |
| In Kahn's topological sort, how do you detect a cycle in the graph? | After BFS completes, if `order.size() != n`, there is a cycle. Nodes in the cycle never reach in-degree 0, so they are never added to the queue and never appear in the result. |
| What is a Dead Letter Queue (DLQ) in Kafka and what problem does it solve? | A DLQ is a separate topic to which messages are routed after N failed processing retries. It solves the poison-pill problem: without a DLQ, a message that always fails blocks all subsequent messages in the same partition, halting the consumer indefinitely. |
