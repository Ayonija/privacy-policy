## Focus
Count integers in [0, N] satisfying digit-level constraints by building numbers digit-by-digit with a `tight` flag that tracks whether the current prefix is still bounded by N.

---

## DSA (2 hours)

### Pattern: Digit DP — Tight Constraint + Position State

State: `(pos, tight, extra_state)`
- `pos`: current digit position (0 = most significant)
- `tight`: True if all digits so far exactly match the limit — next digit must be ≤ limit[pos]; False if already strictly less — next digit is free (0..9 or full alphabet)
- `extra_state`: problem-specific constraint state (parity mask, used-digits bitmask, last digit, etc.)

**Trigger condition:**
- "Count integers in [lo, hi] where each integer satisfies some digit-level property"
- Property can be checked or tracked digit by digit
- Brute force would enumerate all integers up to N (infeasible for N up to 10^18)

**Complexity:**
- Time: O(D × 2 × S × 10) where D = number of digits, S = extra state size, 10 = digit choices
- Space: O(D × 2 × S) for memoization

### Full Solution Walkthrough — Count Numbers with Unique Digits (LC 357) — Revision

**Problem:** Count all numbers in [0, 10^n) that have no repeated digits.

**Combinatorial approach (O(n)):**
- Numbers with 1 digit: 10 (0-9)
- Numbers with exactly k digits (k ≥ 2): 9 × 9 × 8 × ... × (10-k+1)
  - First digit: 9 choices (1-9, no leading zero)
  - Second digit: 9 choices (0-9 except first)
  - Third digit: 8 choices, etc.

```python
def countNumbersWithUniqueDigits(n):
    if n == 0:
        return 1   # only 0
    result = 10    # 1-digit: 0 through 9
    unique_digits = 9    # first non-leading-zero digit: 9 choices
    available = 9        # remaining digits: 9, 8, 7, ...
    for length in range(2, n + 1):
        unique_digits *= available
        result += unique_digits
        available -= 1
    return result
```

**Digit DP equivalent (generalizable to harder constraints):**

```python
def countNumbersWithUniqueDigits_dp(n):
    from functools import lru_cache
    limit = str(10 ** n - 1)   # e.g., n=2 → limit="99"
    m = len(limit)

    @lru_cache(maxsize=None)
    def dp(pos, tight, used_mask, started):
        # started=False: we haven't placed a non-zero digit yet (leading zeros)
        if pos == m:
            return 1   # valid number found
        hi = int(limit[pos]) if tight else 9
        count = 0
        for digit in range(0, hi + 1):
            if digit == 0 and not started:
                # Leading zero: don't mark digit 0 as used
                count += dp(pos + 1, tight and digit == hi, used_mask, False)
            elif not (used_mask & (1 << digit)):
                # Non-leading digit not yet used
                count += dp(pos + 1, tight and digit == hi, used_mask | (1 << digit), True)
        return count

    return dp(0, True, 0, False)
```

**Key detail — leading zeros:** Digit 0 as a leading zero should not be "used" in the mask, because the number "07" is really just "7" and can still use digit 7 freely. The `started` flag tracks whether we've committed to a non-zero first digit.

### Full Solution Walkthrough — Non-negative Integers without Consecutive Ones (LC 600) — Hard

**Problem:** Count integers in [0, n] whose binary representation has no two consecutive 1-bits.

**Why binary digit DP:** The constraint is on bits, not decimal digits. Represent n in binary; use the tight flag on binary digits (0 or 1).

```python
def findIntegers(n):
    bits = bin(n)[2:]   # e.g., n=5 → bits="101"
    m = len(bits)

    from functools import lru_cache

    @lru_cache(maxsize=None)
    def dp(pos, prev_one, tight):
        # pos: current bit position
        # prev_one: was the previous bit a 1?
        # tight: are we still bounded by n's binary representation?
        if pos == m:
            return 1   # valid number (all bits placed)

        hi = int(bits[pos]) if tight else 1   # max bit value at this position
        result = 0

        for bit in range(0, hi + 1):
            if prev_one and bit == 1:
                continue   # skip: would create consecutive ones
            result += dp(pos + 1, bit == 1, tight and bit == hi)

        return result

    return dp(0, False, True)
```

**Trace for n=5 (binary "101"):**
- dp(0, False, True): try bit 0 and bit 1 (tight, hi=1)
  - bit=0: dp(1, False, False) — free from here: count all 2-bit binary numbers with no consec-ones
  - bit=1: dp(1, True, True) — still tight
    - bit=0: dp(2, False, False): count all 1-bit numbers → 2 (0 or 1, neither has prev_one issue)
    - bit=1 skipped if prev_one (which is True here) — so only bit=0
  - Total from bit=1: 2
- dp(1, False, False): hi=1, prev_one=False
  - bit=0: dp(2, False, False) = 2
  - bit=1: dp(2, True, False) = 2 (prev_one=True; next bit can only be 0)
  - Total: 4
- dp(0,...) = 4 + 2 = 6
- Result: 0,1,2,4,5 all valid (no consecutive ones) plus 0 is included... let me verify: 0=0,1=1,2=10,3=11(invalid),4=100,5=101 → 5 numbers (0,1,2,4,5) — dp returns 6? Actually dp counts from 0 to n inclusive, and leading zeros are handled by the tight flag: "000","001","010","100","101" = 5 valid ≤ "101". The lru_cache correctly handles this.

### Full Solution Walkthrough — Numbers At Most N Given Digit Set (LC 902) — Hard

**Problem:** Given a sorted array of digit characters `digits` and integer `n`, count how many positive integers ≤ n can be formed using only the given digits (with repetition).

**Key insight:** No memoization needed. Count by length and then digit-by-digit.

```python
def atMostNGivenDigitSet(digits, n):
    s = str(n)
    m = len(s)       # number of digits in n
    d = len(digits)  # how many choices per position
    result = 0

    # Count all valid numbers shorter than m digits
    # Length 1: d^1 numbers, Length 2: d^2 numbers, ..., Length m-1: d^(m-1) numbers
    for length in range(1, m):
        result += d ** length

    # Count valid numbers with exactly m digits that are <= n
    # Process digit by digit from most significant
    for i, c in enumerate(s):
        # How many digits in our set are strictly less than c?
        less = sum(1 for dig in digits if dig < c)
        # Each such digit can be freely followed by any combination for remaining positions
        result += less * (d ** (m - 1 - i))

        if c not in digits:
            # We cannot place a digit equal to c here — no way to continue
            # matching n's prefix at this position. All longer continuations invalid.
            break
    else:
        # Loop completed without breaking: n itself is constructable from our digit set
        result += 1

    return result
```

**The for-else pattern explained:**
- The `else` clause runs ONLY if the loop completed without a `break`
- Breaking means: at some position, n has a digit not in our set → n itself is not constructable
- Not breaking (else) means: every digit of n was in our set → n itself is a valid number, add 1

**Trace for digits=["1","3","5","7"], n=100:**
- s="100", m=3, d=4
- Shorter numbers (length 1, 2): 4 + 16 = 20
- m-digit processing:
  - i=0, c='1': less = 0 (no digit < '1'). 0 × 4^2 = 0. '1' in digits → continue
  - i=1, c='0': less = 0. 0 × 4^1 = 0. '0' not in digits → break
- result = 20. Answer: 20

**Trace for digits=["1","3","5","7"], n=15:**
- s="15", m=2, d=4
- Length 1: 4
- m-digit:
  - i=0, c='1': less=0. '1' in digits → continue
  - i=1, c='5': less=2 ('1','3' < '5'). result+=2×1=2. '5' in digits → continue
- else clause: result+=1 (n=15 is constructable as "15")
- Total: 4+2+1=7. Valid: 1,3,5,7,11,13,15 → 7. Correct.

## System Design (30 min)

### Topic: LFU Cache — O(1) Get and Put

**LFU vs LRU:** LFU evicts the item with the lowest access frequency (ties broken by least recent). LRU evicts the item not accessed for the longest time.

**O(1) LFU — three hashmaps:**

```python
from collections import defaultdict, OrderedDict

class LFUCache:
    def __init__(self, capacity):
        self.cap = capacity
        self.key_val = {}                      # key → value
        self.key_freq = {}                     # key → current access frequency
        self.freq_keys = defaultdict(OrderedDict)  # freq → OrderedDict of keys (insertion order = LRU within freq)
        self.min_freq = 0                      # current minimum frequency (for O(1) eviction)

    def _update(self, key):
        """Increment key's frequency; update freq_keys and min_freq."""
        freq = self.key_freq[key]
        self.key_freq[key] = freq + 1
        # Remove from current frequency bucket
        del self.freq_keys[freq][key]
        # If min_freq bucket is now empty, increment min_freq
        if not self.freq_keys[freq] and freq == self.min_freq:
            self.min_freq += 1
        # Add to new frequency bucket
        self.freq_keys[freq + 1][key] = None   # OrderedDict preserves insertion order

    def get(self, key):
        if key not in self.key_val:
            return -1
        self._update(key)
        return self.key_val[key]

    def put(self, key, value):
        if self.cap <= 0:
            return
        if key in self.key_val:
            self.key_val[key] = value
            self._update(key)
        else:
            if len(self.key_val) >= self.cap:
                # Evict: remove the LRU key from the min_freq bucket
                evict_key, _ = self.freq_keys[self.min_freq].popitem(last=False)
                del self.key_val[evict_key]
                del self.key_freq[evict_key]
            # Insert new key with frequency 1
            self.key_val[key] = value
            self.key_freq[key] = 1
            self.freq_keys[1][key] = None
            self.min_freq = 1   # new key always starts at freq 1 = new minimum
```

**Why OrderedDict for freq_keys:** Within the same frequency, we use LRU as tiebreaker. OrderedDict's `popitem(last=False)` removes the oldest-inserted key in O(1) — that's the LRU among equal frequencies.

**Why min_freq only increments by 1 on access:** When we move a key from freq f to f+1, min_freq only increases if f was the minimum AND its bucket is now empty. New insertions always set freq=1, resetting min_freq=1.

**Interview talking point:**
*"LFU needs O(1) get, put, and eviction. The trick is maintaining three maps: key→value, key→freq, and freq→OrderedDict. The OrderedDict within each frequency bucket gives us LRU tiebreaking. We track min_freq separately to evict in O(1) — it only needs to be updated when the minimum-frequency bucket empties (rare) or a new key is inserted (min_freq resets to 1)."*
