## DSA (2 hours)

### Pattern: TSP-style Bitmask DP

State: `dp[mask][last]` = optimal value (min cost / min length) when the set of visited nodes is `mask` and the last node visited is `last`.

Transition: for each unvisited node `nxt` not in `mask`:
`dp[mask | (1<<nxt)][nxt] = dp[mask][last] + cost(last, nxt)`

**Trigger condition:**
- "Visit all n items in some order, minimize/maximize total cost"
- n ≤ 20 (bitmask feasible)
- TSP, Hamiltonian path, shortest superstring, minimum spanning permutation

**Complexity:**
- Time: O(2^n × n²) — 2^n states × n last-nodes × n transitions
- Space: O(2^n × n) for dp table + parent table for reconstruction

### Full Solution Walkthrough — Shortest Path Visiting All Nodes (LC 847) — Revision

**Problem:** Find the shortest path that visits every node in an undirected graph (can revisit nodes and edges). Return the number of edges in the shortest such path.

**Why BFS, not DP:** The graph is unweighted. BFS explores states layer by layer (each layer = one more edge). The first time a state (node, full_mask) is reached is guaranteed to be via the shortest path.

```python
def shortestPathLength(graph):
    from collections import deque
    n = len(graph)
    full = (1 << n) - 1

    # Initialize: start BFS from ALL nodes simultaneously
    # Each starting node seeds a state with only itself visited
    queue = deque()
    visited = set()

    for i in range(n):
        initial_mask = 1 << i
        queue.append((i, initial_mask, 0))   # (node, visited_mask, distance)
        visited.add((i, initial_mask))

    while queue:
        node, mask, dist = queue.popleft()

        if mask == full:
            return dist   # first time all nodes visited = shortest

        for nei in graph[node]:
            new_mask = mask | (1 << nei)
            if (nei, new_mask) not in visited:
                visited.add((nei, new_mask))
                queue.append((nei, new_mask, dist + 1))

    return -1   # unreachable (connected graph always returns above)
```

**Why start from all nodes:** The problem asks for the shortest path that visits all nodes, starting from any node. Starting BFS from all nodes simultaneously is equivalent to asking "which starting node leads to the minimum steps?"

**State space size:** n × 2^n states. For n=12: 12 × 4096 = 49,152 states — very manageable for BFS.

### Full Solution Walkthrough — Find the Shortest Superstring (LC 943) — Hard

**Problem:** Given an array of unique words, find the shortest string that contains each word as a substring. No word is a prefix of another.

**Key precomputation:** `overlap[i][j]` = length of the longest suffix of `words[i]` that equals a prefix of `words[j]`. When we concatenate word j after word i in the superstring, we save `overlap[i][j]` characters.

```python
def shortestSuperstring(words):
    n = len(words)

    # Precompute overlap[i][j] = suffix of words[i] that matches prefix of words[j]
    overlap = [[0] * n for _ in range(n)]
    for i in range(n):
        for j in range(n):
            if i == j:
                continue
            # Try all possible overlap lengths from longest to shortest
            limit = min(len(words[i]), len(words[j]))
            for k in range(limit, 0, -1):
                if words[i].endswith(words[j][:k]):
                    overlap[i][j] = k
                    break

    INF = float('inf')
    # dp[mask][last] = min total superstring length using words in mask, ending with word `last`
    dp = [[INF] * n for _ in range(1 << n)]
    parent = [[-1] * n for _ in range(1 << n)]

    # Base case: superstring consisting of just word i
    for i in range(n):
        dp[1 << i][i] = len(words[i])

    for mask in range(1, 1 << n):
        for last in range(n):
            if not (mask & (1 << last)):
                continue   # last not in current mask
            if dp[mask][last] == INF:
                continue   # unreachable state

            for nxt in range(n):
                if mask & (1 << nxt):
                    continue   # nxt already in mask
                new_mask = mask | (1 << nxt)
                # Appending words[nxt] after words[last], saving overlap[last][nxt] chars
                new_len = dp[mask][last] + len(words[nxt]) - overlap[last][nxt]
                if new_len < dp[new_mask][nxt]:
                    dp[new_mask][nxt] = new_len
                    parent[new_mask][nxt] = last

    # Find the best ending word
    full = (1 << n) - 1
    last = min(range(n), key=lambda i: dp[full][i])

    # Reconstruct path via parent pointers
    path = []
    mask = full
    while last != -1:
        path.append(last)
        prev = parent[mask][last]
        mask ^= (1 << last)
        last = prev
    path.reverse()

    # Build the actual superstring
    result = words[path[0]]
    for k in range(1, len(path)):
        ov = overlap[path[k - 1]][path[k]]
        result += words[path[k]][ov:]   # skip the overlapping prefix

    return result
```

**Example trace for ["ab","ba","aba"]:**
- overlap[0][1]=1 ("ab" ends with "b"="ba"[:1]), overlap[1][0]=1, overlap[0][2]=2, overlap[2][0]=2, overlap[1][2]=1, overlap[2][1]=2
- dp[001][0]=2, dp[010][1]=2, dp[100][2]=3
- dp[011][1] = dp[001][0] + len("ba") - 1 = 2+2-1=3
- dp[101][2] = dp[001][0] + len("aba") - 2 = 2+3-2=3
- ... dp[111][2] ends up as min → "aba" (length 3) contains all three

### Full Solution Walkthrough — Distribute Repeating Integers (LC 1655) — Hard

**Problem:** Given `nums` (multiset of integers), and `quantity[i]` units needed by customer i, can you satisfy all customers? Each customer must receive the same integer value.

**Approach:** Count distinct value frequencies. dp[mask] = can we satisfy the customers in `mask` using some prefix of the frequency counts.

```python
def canDistribute(nums, quantity):
    from collections import Counter
    cnt = list(Counter(nums).values())   # available supply of each distinct value
    m = len(quantity)

    # Precompute need[mask] = total quantity needed by customers in mask
    need = [0] * (1 << m)
    for mask in range(1, 1 << m):
        lsb = mask & (-mask)             # isolate lowest set bit
        i = lsb.bit_length() - 1         # index of that customer
        need[mask] = need[mask ^ lsb] + quantity[i]

    dp = [False] * (1 << m)
    dp[0] = True   # satisfying no customers is trivially possible

    for supply in cnt:
        # Process each supply count once (0/1 decision: use this supply or not)
        # Iterate BACKWARDS to avoid using the same supply count twice
        for mask in range((1 << m) - 1, -1, -1):
            if not dp[mask]:
                continue
            # Try assigning a subset of the remaining (unsatisfied) customers to this supply
            complement = ((1 << m) - 1) ^ mask   # customers not yet satisfied
            sub = complement
            while sub > 0:
                if need[sub] <= supply:
                    dp[mask | sub] = True
                sub = (sub - 1) & complement

    return dp[(1 << m) - 1]
```

**Why iterate masks backwards:** Same principle as 0/1 knapsack. When we update `dp[mask | sub] = True`, we don't want to use this result in the same outer loop iteration (which would allow using the same supply value twice). Backwards iteration ensures `dp[mask]` was set in a PREVIOUS supply's iteration.

**Complexity:** O(|cnt| × 3^m). For m=10 customers and up to 50 distinct values: 50 × 3^10 = ~2.95M operations. Feasible.

## System Design (30 min)

### Topic: Consistent Hashing — Distributed Cache Cluster Design

**Redis Cluster vs. Consistent Hashing:**

| Aspect | Redis Cluster | Consistent Hashing Ring |
|--------|--------------|------------------------|
| Slot model | 16384 fixed hash slots | Continuous ring [0, 2^32) |
| Key → slot | CRC16(key) % 16384 | hash(key), first clockwise node |
| Resharding | Manual (redis-cli --cluster reshard) | Automatic (only successor keys move) |
| Client routing | Clients get MOVED redirect on wrong node | Client computes node locally |
| Replication | Each slot has primary + replicas | Depends on implementation |

**Hash slot assignment example:**
- Node A: slots 0–5460
- Node B: slots 5461–10922
- Node C: slots 10923–16383
- Adding Node D: move some ranges from A, B, C to D (manual)

**Replication in Redis Cluster:**
- Each primary owns a shard; replicas async replicate writes
- Failover: if primary unreachable for `cluster-node-timeout` (default 15s), its replica becomes primary
- Risk: async replication → writes to primary that haven't replicated to replica can be lost during failover
- Mitigation: `WAIT numreplicas timeout` — blocks until numreplicas acknowledge the write

**Gossip protocol:** Nodes exchange state every 100ms via gossip. Cluster state (which nodes are alive, slot assignments) propagates quickly without a central coordinator.

**Interview talking point:**
*"Redis Cluster uses 16384 fixed hash slots instead of a consistent hash ring. This makes resharding explicit — you decide which slots move between nodes. For a cache cluster at 10M QPS, I'd start with 6 nodes (3 primary + 3 replica), use hash tags {user_id} to collocate related keys on the same shard, and monitor cluster_stats_messages_sent to detect gossip storms."*
