## DSA (2 hours)

### Pattern: Advanced 2D String DP and Interval DP on Strings

**Core idea:**
Regular Expression Matching extends 2D string alignment DP with special state transitions for `a*` patterns (zero or more). Edit Distance uses a unified cost-per-edit framework. Scramble String uses memoized recursion (interval DP) where the state is a pair of substrings.

**Trigger condition:**
- `.` and `*` in pattern → Regular Expression DP
- Insert / delete / replace cost → Edit Distance DP
- "Is B a scrambled form of A (recursive character swaps)?" → memoized recursion with anagram pruning

**Key insight — `a*` in regex:**
`p[j-1] == '*'` means "zero or more of `p[j-2]`". Two choices:
1. Use zero occurrences of `p[j-2]`: `dp[i][j] = dp[i][j-2]` (delete the `a*` pattern)
2. Use one or more: only if `p[j-2]` matches `s[i-1]` (via `.` or literal): `dp[i][j] |= dp[i-1][j]`

### Full Solution Walkthrough — Edit Distance (LC 72) — Revision

**Problem:** Minimum insert, delete, or replace operations to convert `word1` to `word2`.

**State:** `dp[i][j]` = min operations to convert `word1[0..i-1]` to `word2[0..j-1]`.

**Recurrence:**
```
If word1[i-1] == word2[j-1]:  dp[i][j] = dp[i-1][j-1]            (no operation)
Else:
    dp[i][j] = 1 + min(
        dp[i-1][j-1],   # replace word1[i-1] with word2[j-1]
        dp[i-1][j],     # delete word1[i-1]
        dp[i][j-1],     # insert word2[j-1] after word1[i-1]
    )
```

**Base:** `dp[i][0] = i` (delete i chars from word1), `dp[0][j] = j` (insert j chars).

```python
def minDistance(word1: str, word2: str) -> int:
    m, n = len(word1), len(word2)
    dp = list(range(n + 1))  # dp[0][j] = j

    for i in range(1, m + 1):
        prev = dp[0]       # saves dp[i-1][j-1] before overwrite
        dp[0] = i          # dp[i][0] = i
        for j in range(1, n + 1):
            temp = dp[j]   # save dp[i-1][j] before overwrite
            if word1[i - 1] == word2[j - 1]:
                dp[j] = prev
            else:
                dp[j] = 1 + min(prev, dp[j], dp[j - 1])
            prev = temp

    return dp[n]
```

**Trace on word1="horse", word2="ros":**
```
Base row: [0,1,2,3]
After h:  [1,1,2,3]
After o:  [2,2,1,2]
After r:  [3,2,2,2]
After s:  [4,3,3,2]
After e:  [5,4,4,3]  → answer = 3
```

### Full Solution Walkthrough — Regular Expression Matching (LC 10) — Hard

**Problem:** Match string `s` against pattern `p` with `.` (any single char) and `*` (zero or more of preceding char).

**Key difference from LC 44 (Wildcard):** Here `*` always refers to the preceding character `p[j-2]`. Pattern `a*` means "zero or more 'a's". Pattern `.*` means "zero or more of any char".

**State:** `dp[i][j]` = True if `s[0..i-1]` matches `p[0..j-1]`.

**Recurrence:**
```
If p[j-1] == '*':
    # Zero occurrences: eliminate p[j-2] and p[j-1]
    dp[i][j] = dp[i][j-2]
    # One or more: if p[j-2] matches s[i-1]
    if p[j-2] == '.' or p[j-2] == s[i-1]:
        dp[i][j] |= dp[i-1][j]   # consume s[i-1], keep using p[j-2]*
Elif p[j-1] == '.' or p[j-1] == s[i-1]:
    dp[i][j] = dp[i-1][j-1]
# else: dp[i][j] = False
```

**Base cases:**
- `dp[0][0] = True`
- `dp[0][j] = True` if p[0..j-1] can match empty string → only if `p[j-1]=='*'` and `dp[0][j-2]` is True (the `a*` eliminates itself)

```python
def isMatch(s: str, p: str) -> bool:
    m, n = len(s), len(p)
    dp = [[False] * (n + 1) for _ in range(m + 1)]
    dp[0][0] = True

    # Patterns like a*, a*b*, a*b*c* can match empty s
    for j in range(2, n + 1):
        if p[j - 1] == '*':
            dp[0][j] = dp[0][j - 2]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if p[j - 1] == '*':
                dp[i][j] = dp[i][j - 2]  # zero occurrences
                if p[j - 2] == '.' or p[j - 2] == s[i - 1]:
                    dp[i][j] |= dp[i - 1][j]  # one or more
            elif p[j - 1] == '.' or p[j - 1] == s[i - 1]:
                dp[i][j] = dp[i - 1][j - 1]

    return dp[m][n]
```

**Trace on s="aab", p="c*a*b":**
- p = ['c','*','a','*','b'], n=5
- dp[0]: [T,F,T,F,T,F] — c* matches empty, a* matches empty, but b≠*
  - dp[0][2]=dp[0][0]=T (c* eliminated), dp[0][4]=dp[0][2]=T (a* eliminated)
- dp[3][5]: p[4]='b', s[2]='b' → dp[2][4]. Check dp[2][4]: p[3]='*', p[2]='a', s[1]='a' → dp[2][2] or dp[1][4]... eventually True ✓

### Full Solution Walkthrough — Scramble String (LC 87) — Hard

**Problem:** s1 is scrambled into s2 via: recursively split s1 at some position, possibly swap the two halves, and scramble each half. Return whether s2 is a scramble of s1.

**Memoized recursion:** `dp(a, b)` = True if b is a scramble of a.

**Base:** if `a == b` → True. If anagrams differ → False (fast pruning).

**Recurrence:** Try all split points i from 1 to n-1:
- No swap: `dp(a[:i], b[:i]) and dp(a[i:], b[i:])`
- Swap: `dp(a[:i], b[n-i:]) and dp(a[i:], b[:n-i])`

```python
def isScramble(s1: str, s2: str) -> bool:
    from functools import lru_cache

    @lru_cache(maxsize=None)
    def dp(a: str, b: str) -> bool:
        if a == b:
            return True
        if sorted(a) != sorted(b):  # anagram check: necessary condition
            return False
        n = len(a)
        for i in range(1, n):
            # No swap: left parts align, right parts align
            if dp(a[:i], b[:i]) and dp(a[i:], b[i:]):
                return True
            # Swap: left of a aligns with right of b, right of a aligns with left of b
            if dp(a[:i], b[n - i:]) and dp(a[i:], b[:n - i]):
                return True
        return False

    return dp(s1, s2)
```

**Complexity:** O(n⁴) states (all substrings of s1 and s2) × O(n) splits = O(n⁵) worst case without the anagram pruning. Anagram check prunes ~90% of branches in practice.

**Why memoize on strings?** The same pair (a, b) can appear from different split paths. Memoization ensures each unique (a, b) pair is computed once.

## System Design (30 min)

### Topic: Rate Limiting — Sliding Window Counters and Distributed Rate Limiting

**Fixed Window Counter (simplest):**
- Increment counter for current window `[minute_start, minute_end]`; reset at window boundary
- Problem: **boundary burst** — a user can make 2× limit requests by sending limit/2 before boundary and limit/2 after

**Sliding Window Counter (approximate, O(1)):**
- Keep two windows: previous and current
- Estimate count in sliding window: `current + previous × (overlap_fraction)`
- `overlap_fraction` = fraction of previous window still in the sliding window
- 1 Redis HASH with two fields. O(1) per request. ≈1% error at most.

**Sliding Window Log (exact, O(requests)):**
- Store each request timestamp in a Redis Sorted Set (score = timestamp)
- On request:
  1. `ZREMRANGEBYSCORE key 0 (now - window_ms)` — remove old entries
  2. `ZADD key now now` — add current request
  3. `count = ZCARD key` — count in window
  4. If count > limit → reject
- O(log n) per request. Exact but memory-intensive.

**Distributed Rate Limiting at Scale:**
- Local cache + global sync: allow some burst locally, sync periodically to global Redis
- Problem: 1M users × per-user Redis key = high memory usage → use TTL aggressively
- Sharding: shard rate limit state by user_id to distribute Redis load
- Interview decision: "For FAANG-scale APIs, use a hybrid: local in-process token bucket with periodic sync to Redis. Pure Redis per-request is the bottleneck at 100K+ RPS."

**Interview talking point:**
*"The sliding window log is exact but memory-intensive — each request stores a timestamp. The sliding window counter approximation stores only 2 integers per user and is within 1% of the exact count, making it the practical choice. For multi-server deployments, use Redis so all servers share the same rate limit state."*
