## DSA (2 hours)

### Pattern: Two-Agent Simultaneous DP on Grids

**Core idea:**
When two agents make decisions simultaneously and their paths interact (e.g., they can't both collect cherries from the same cell), model both agents in a single DP state. Key invariant: both agents always take the same number of steps, so parameterize by step count t and one coordinate per agent; the other coordinate is derived.

**Trigger condition:**
- Two people collecting resources on a shared grid — can't double-count shared cells
- "Go from start to end and back" — equivalent to two forward-only paths
- State space: O(n³) for 2 agents on n×n grid (row1, step, row2)

**Complexity:**
O(n³) time and space | Memoized recursion often cleaner than bottom-up for two-agent problems

### Full Solution Walkthrough — Last Stone Weight II (LC 1049) — Revision

**Problem:** Smash stones together. At end, minimize remaining stone weight.

**Key insight:** Every stone ends up in one of two groups (positive or negative sign). Minimize |S1 − S2| subject to S1 + S2 = total. Equivalent to: find max subset sum ≤ total // 2; answer = total − 2 × that subset sum.

```python
def lastStoneWeightII(stones: list[int]) -> int:
    total = sum(stones)
    target = total // 2

    dp = [False] * (target + 1)
    dp[0] = True

    for stone in stones:
        for j in range(target, stone - 1, -1):
            dp[j] |= dp[j - stone]

    # Find largest achievable sum ≤ target
    for j in range(target, -1, -1):
        if dp[j]:
            return total - 2 * j

    return total  # fallback (shouldn't be reached)
```

**Why `total − 2 × j`?** If S1 = j and S2 = total − j, then |S1 − S2| = |2j − total| = total − 2j (since j ≤ total//2).

### Full Solution Walkthrough — Cherry Pickup (LC 741) — Hard

**Problem:** n×n grid with cherries (0/1) and thorns (-1). Start at (0,0), go to (n-1,n-1), come back. Maximize cherries collected. Thorns block path.

**Two-agent equivalence:** One round trip = two agents going simultaneously from (0,0) to (n-1,n-1). After t steps, agent 1 is at (r1, t-r1) and agent 2 is at (r2, t-r2). We only need to track (r1, r2) given t.

**State:** `dp(r1, c1, r2)` where c1 is derived and c2 = (r1+c1) - r2.
Both agents move right or down each step.

```python
def cherryPickup(grid: list[list[int]]) -> int:
    n = len(grid)
    from functools import lru_cache

    @lru_cache(maxsize=None)
    def dp(r1: int, c1: int, r2: int) -> int:
        c2 = r1 + c1 - r2  # derived: both agents have taken r1+c1 steps
        # Out of bounds
        if r1 >= n or c1 >= n or r2 >= n or c2 >= n:
            return float('-inf')
        # Thorn check
        if grid[r1][c1] == -1 or grid[r2][c2] == -1:
            return float('-inf')
        # Reached the end
        if r1 == n - 1 and c1 == n - 1:
            return grid[r1][c1]

        # Cherries at current positions (avoid double-counting)
        cherries = grid[r1][c1]
        if not (r1 == r2 and c1 == c2):  # different cells
            cherries += grid[r2][c2]

        # Both agents each independently move right or down → 4 combinations
        best = max(
            dp(r1 + 1, c1, r2 + 1),   # both down
            dp(r1 + 1, c1, r2),        # agent1 down, agent2 right
            dp(r1, c1 + 1, r2 + 1),   # agent1 right, agent2 down
            dp(r1, c1 + 1, r2),        # both right
        )

        return cherries + best

    result = dp(0, 0, 0)
    return max(0, result)
```

**Why max(0, result)?** If no valid path exists (all paths blocked), dp returns −∞. Clamp to 0.

**Complexity:** O(n³) states (r1, c1, r2 with c2 derived), O(1) work per state.

### Full Solution Walkthrough — Cherry Pickup II (LC 1463) — Hard

**Problem:** Robot 1 starts at (0, 0) and Robot 2 starts at (0, cols-1) of a grid. Both move down one row per step. Each can move diagonally left, straight down, or diagonally right. Maximize total cherries (don't double-count same cell).

**State:** `dp(row, c1, c2)` = max cherries from row to end when robot 1 is at column c1, robot 2 at c2.

```python
def cherryPickup(grid: list[list[int]]) -> int:
    rows, cols = len(grid), len(grid[0])
    from functools import lru_cache

    @lru_cache(maxsize=None)
    def dp(row: int, c1: int, c2: int) -> int:
        if row == rows:
            return 0

        cherries = grid[row][c1] + (grid[row][c2] if c1 != c2 else 0)

        best = 0
        for dc1 in [-1, 0, 1]:
            for dc2 in [-1, 0, 1]:
                nc1, nc2 = c1 + dc1, c2 + dc2
                if 0 <= nc1 < cols and 0 <= nc2 < cols:
                    best = max(best, dp(row + 1, nc1, nc2))

        return cherries + best

    return dp(0, 0, cols - 1)
```

**Complexity:** O(rows × cols²) states, O(9) = O(1) transitions each. Total O(rows × cols²).

**Contrast with Cherry Pickup I:** In II, the column of each robot is independent (no shared step counter trick needed) — the state is simply (row, c1, c2). In I, columns are derived from step count.

## System Design (30 min)

### Topic: GraphQL — Schema, Resolvers, and the N+1 Problem

**GraphQL Schema (SDL):**
```graphql
type User {
  id: ID!
  name: String!
  posts: [Post!]!     # this field is resolved separately
}

type Post {
  id: ID!
  title: String!
  author: User!
}

type Query {
  users: [User!]!
  user(id: ID!): User
}
```

**Resolver execution model:**
- Each field can have a resolver function. If missing, default resolver uses parent object's property.
- Query for `users { name posts { title } }` executes:
  1. `Query.users` → returns list of N users
  2. For each of N users: `User.posts` → N separate DB calls (the N+1 problem!)

**The N+1 Problem:**
```
Naive:
GET /graphql → resolver calls users() → 1 query
For each user → resolver calls posts(userId) → N queries
Total: N+1 queries for N users
```

**DataLoader Solution:**
DataLoader batches all `.load(userId)` calls made within the same event loop tick into a single call:
```javascript
const postLoader = new DataLoader(async (userIds) => {
    const posts = await db.query('SELECT * FROM posts WHERE user_id IN (?)', [userIds]);
    // Group posts by userId and return in same order as userIds
    return userIds.map(id => posts.filter(p => p.userId === id));
});

// In User.posts resolver:
posts: (parent) => postLoader.load(parent.id)  // batched automatically
```
Total: 2 queries regardless of N users.

**Interview talking point:**
*"The N+1 problem is the most common GraphQL performance mistake. DataLoader solves it by batching all loads within a single event loop tick. Key requirements: (1) the batch function receives an array of keys and returns an array of values in the same order; (2) DataLoader per-request (not shared across requests) to avoid cross-user data leakage."*

## Flashcards

| Q | A |
|---|---|
| In Cherry Pickup (LC 741), why can two agents simultaneously going (0,0)→(n-1,n-1) model a round trip? | Going to (n-1,n-1) and back = two non-intersecting paths in opposite directions. By reversing the return trip to also go (0,0)→(n-1,n-1), we model both as forward paths simultaneously, sharing the cherry-collection constraint. |
| In Cherry Pickup (LC 741), how do you derive c2 from (r1, c1, r2)? | Both agents have taken the same number of steps = r1+c1 = r2+c2. Therefore c2 = r1+c1-r2. This reduces the state from 4D to 3D. |
| What are the 9 transitions in Cherry Pickup II? | Each robot independently moves left (dc=-1), stays (dc=0), or right (dc=+1). 3 choices × 3 choices = 9 combinations. Filter out-of-bounds before recursing. |
| How does Last Stone Weight II reduce to a knapsack problem? | Answer = total − 2 × (max subset sum ≤ total//2). Find the subset summing closest to total//2 from below using 0/1 boolean knapsack. |
| What is DataLoader and what problem does it solve in GraphQL? | DataLoader batches all `.load(key)` calls within one event loop tick into a single batch function call. It solves the N+1 problem: instead of N queries for N parent objects, it makes 1 batch query. Must be instantiated per request to avoid data leakage. |
