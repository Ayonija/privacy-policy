## DSA (2 hours)

### Pattern: LCS — 2D String Alignment DP

**Core idea:**
Align two strings character by character. `dp[i][j]` represents the answer for the prefixes `s1[0..i-1]` and `s2[0..j-1]`.

**The fundamental LCS recurrence:**
```
If s1[i-1] == s2[j-1]:  dp[i][j] = dp[i-1][j-1] + 1      (characters match → extend)
Else:                    dp[i][j] = max(dp[i-1][j], dp[i][j-1])  (skip one character)
```

**Trigger condition:**
- "Longest common subsequence / substring"
- "Minimum edits to transform string A to B"
- "Count occurrences of pattern t in string s as subsequences"
- "Do strings interleave to form target?"
- Always 2D DP over the two string lengths; fill row by row

**Complexity:** O(mn) time | O(min(m,n)) space with 1D rolling array

**LCS → many reductions:**
- Minimum insertions to palindrome: LCS(s, reverse(s))
- Shortest Common Supersequence: m + n − LCS(s1, s2)
- Edit distance: related but counts substitutions, not just insertions/deletions

### Full Solution Walkthrough — Longest Common Subsequence (LC 1143) — Revision

```python
def longestCommonSubsequence(text1: str, text2: str) -> int:
    m, n = len(text1), len(text2)
    # dp[i][j] = LCS of text1[0..i-1] and text2[0..j-1]
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i - 1] == text2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    return dp[m][n]
```

**Space optimization to O(n):** Maintain only the previous row. When updating `dp[j]`, save `dp[i-1][j-1]` in a variable `prev` before overwriting.

```python
def longestCommonSubsequence(text1: str, text2: str) -> int:
    m, n = len(text1), len(text2)
    dp = [0] * (n + 1)

    for i in range(1, m + 1):
        prev = 0  # represents dp[i-1][j-1]
        for j in range(1, n + 1):
            temp = dp[j]  # save before overwriting
            if text1[i - 1] == text2[j - 1]:
                dp[j] = prev + 1
            else:
                dp[j] = max(dp[j], dp[j - 1])
            prev = temp

    return dp[n]
```

### Full Solution Walkthrough — Distinct Subsequences (LC 115) — Hard

**Problem:** Count distinct ways to form string `t` as a subsequence of string `s`.

**State:** `dp[i][j]` = number of ways to form `t[0..j-1]` from `s[0..i-1]`.

**Recurrence:**
- Always include "skip `s[i-1]`": `dp[i][j] = dp[i-1][j]`
- If `s[i-1] == t[j-1]`: also add "use `s[i-1]`": `dp[i][j] += dp[i-1][j-1]`

**Base:** `dp[i][0] = 1` for all i — one way to form empty t (choose nothing).

```python
def numDistinct(s: str, t: str) -> int:
    m, n = len(s), len(t)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(m + 1):
        dp[i][0] = 1  # empty t: always 1 way

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            dp[i][j] = dp[i - 1][j]  # skip s[i-1]
            if s[i - 1] == t[j - 1]:
                dp[i][j] += dp[i - 1][j - 1]  # use s[i-1]

    return dp[m][n]
```

**Trace on s="rabbbit", t="rabbit":**
- When s[i-1]=='b' and t[j-1]=='b': we can use any of the three 'b's → counts multiply
- Final dp[7][6] = 3 (three ways to pick which 'b' to skip)

**Key contrast with LCS:** LCS takes max (best choice); Distinct Subsequences adds (count all choices).

### Full Solution Walkthrough — Wildcard Matching (LC 44) — Hard

**Problem:** Does string `s` match pattern `p`? Pattern has `?` (any single char) and `*` (any sequence, including empty).

**State:** `dp[i][j]` = True if `s[0..i-1]` matches `p[0..j-1]`.

**Recurrence:**
```
If p[j-1] == '*':
    dp[i][j] = dp[i][j-1]       (use * to match zero chars from s)
             OR dp[i-1][j]      (use * to match one more char from s)
Elif p[j-1] == '?' or p[j-1] == s[i-1]:
    dp[i][j] = dp[i-1][j-1]    (both advance one char)
Else:
    dp[i][j] = False
```

**Base cases:**
- `dp[0][0] = True` — empty s matches empty p
- `dp[0][j] = True` only if all of p[0..j-1] are `*` (they can match empty s)

```python
def isMatch(s: str, p: str) -> bool:
    m, n = len(s), len(p)
    dp = [[False] * (n + 1) for _ in range(m + 1)]
    dp[0][0] = True

    # Empty s can match p only if p is all *'s
    for j in range(1, n + 1):
        if p[j - 1] == '*':
            dp[0][j] = dp[0][j - 1]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if p[j - 1] == '*':
                dp[i][j] = dp[i][j - 1] or dp[i - 1][j]
            elif p[j - 1] == '?' or p[j - 1] == s[i - 1]:
                dp[i][j] = dp[i - 1][j - 1]
            # else: remains False

    return dp[m][n]
```

**Trace on s="aa", p="a*":**
```
     ""   a    *
""   T    F    T   (p[1]='*' → dp[0][2] = dp[0][1] = False... wait)
```

Actually for s="aa", p="a*": p[0]='a', p[1]='*'.
- dp[0][0]=T, dp[0][1]=F (p[0]='a'≠'*'), dp[0][2]=T (p[1]='*', dp[0][1]=F... but dp[0][2]=dp[0][1]=F)

Hmm, let me retrace. p="a*" means 'a' followed by '*'. dp[0][2] = dp[0][1] = False. So empty string doesn't match "a*". But "a*" should match "aa" right? Yes: 'a' matches s[0]='a', then '*' matches s[1]='a'. Let me retrace:

dp[1][1]: p[0]='a', s[0]='a' → match → dp[0][0] = T. So dp[1][1]=T.
dp[1][2]: p[1]='*' → dp[1][1] OR dp[0][2] = T OR F = T.
dp[2][2]: p[1]='*' → dp[2][1] OR dp[1][2]. dp[2][1]: p[0]='a', s[1]='a' → dp[1][0] = F. So dp[2][2] = F OR T = T. ✓

**Contrast with LC 10 (Regular Expression):** In LC 10, `*` means "zero or more of the preceding character" — the preceding character matters. In LC 44, `*` means any sequence — no preceding character dependency.

## System Design (30 min)

### Topic: Rate Limiting — Token Bucket and Leaky Bucket Algorithms

**Why rate limiting?**
Protect backend services from abuse, ensure fair usage, prevent DoS/DDoS impact, and enforce API pricing tiers.

**Token Bucket Algorithm:**
- Bucket holds up to `capacity` tokens. Tokens refill at rate `r` tokens/second.
- Each request consumes 1 token. If bucket empty → reject (or queue).
- Allows bursting up to `capacity` requests at once, then sustained at rate `r`.
- **Implementation:** Store `(tokens, last_refill_timestamp)` in Redis. On each request:
  1. Compute tokens accumulated since last refill: `new_tokens = (now - last) * r`
  2. `tokens = min(capacity, tokens + new_tokens)`
  3. If `tokens >= 1`: `tokens -= 1`, allow. Else: reject.

**Leaky Bucket Algorithm:**
- Requests enter a FIFO queue (the "bucket"). Worker drains queue at fixed rate.
- Smooths out bursts — output is always uniform rate.
- Queue full → reject. Never allows bursting.
- **Implementation:** Queue per user in Redis. Rate-limited worker processes queue entries.

**Comparison:**

| | Token Bucket | Leaky Bucket |
|--|-------------|-------------|
| Burst handling | Allows burst up to capacity | No bursting — smooth output |
| Output pattern | Variable (burst then throttle) | Constant rate |
| Use case | APIs where burst is OK | Streaming, audio/video |
| Implementation | Simpler (store tokens + timestamp) | Requires queue |

**Interview talking point:**
*"Token bucket is the most common choice for API rate limiting because it allows clients to burst briefly, which is the natural usage pattern (a user does several actions then pauses). Store `(tokens, last_refill_time)` in Redis; use a Lua script to make the check-and-decrement atomic. Leaky bucket is better for smoothing traffic into a downstream service that can't handle spikes."*
