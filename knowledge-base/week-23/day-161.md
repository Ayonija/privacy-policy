## DSA (1.5 hours — OA Contest Mode)

### Pattern: Sliding Window + Binary Search on Sorted Array
- Core idea: Sort the array and remove duplicates to create a candidate set. For each possible left boundary of a window of size n, binary search for the rightmost element that fits within the range [left, left+n-1]. The number of elements already in that window is the count you keep; the rest (n - window_size) must be replaced via operations. Minimize operations across all valid left positions.
- Trigger condition: Use this pattern when the problem asks for the minimum number of changes to make an array satisfy a contiguous-range property, especially when you can reframe the solution as "how many elements can I keep unchanged."
- Time complexity: O(N log N) | Space complexity: O(N)

### Contest Template (write from memory before solving):
```java
// Sliding window with binary search on sorted deduplicated array
int[] sorted = /* sort + deduplicate original array */;
int n = /* target window size (original array length) */;
int maxKeep = 0;
for (int i = 0; i < sorted.length; i++) {
    // binary search: rightmost index where sorted[j] <= sorted[i] + n - 1
    int lo = i, hi = sorted.length - 1, j = i;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (sorted[mid] <= sorted[i] + n - 1) { j = mid; lo = mid + 1; }
        else hi = mid - 1;
    }
    maxKeep = Math.max(maxKeep, j - i + 1);
}
// answer = n - maxKeep
```

    public List<Integer> findDuplicates(int[] nums) {
        List<Integer> result = new ArrayList<>();
        for (int num : nums) {
            int idx = Math.abs(num) - 1;
            if (nums[idx] < 0) result.add(idx + 1); // visited before → duplicate
            else nums[idx] = -nums[idx];             // mark as visited
        }
        return result;

#### Design Patterns Used
- **Singleton** — `ParkingLot`: only one lot instance exists; `getInstance()` is thread-safe via double-checked locking or enum singleton.
- **Strategy** — `FeeCalculator`: the fee algorithm is swapped at runtime; `ParkingLot` holds a reference to any `FeeCalculator` implementation without knowing the concrete type.
- **Factory** — `VehicleFactory.create(VehicleType, String)`: decouples client code from concrete `Vehicle` subtypes; easy to add new vehicle types without changing callers.

#### Key Trade-off
Concurrent slot reservation: two users submit requests at the same millisecond for the last available slot. Using optimistic locking, the `ParkingSlot` row has an integer `version` column. The UPDATE query reads `UPDATE parking_slot SET is_occupied=true, version=version+1 WHERE slot_id=? AND version=?`. If 0 rows are updated (version mismatch), the application retries by fetching the next available slot. This avoids holding a database lock for the full duration of the request and scales well under moderate contention.

**Interview talking point:** "If asked how to handle simultaneous reservations for the last available slot, answer: use a database row-level lock — SELECT the slot FOR UPDATE inside a transaction; only one transaction acquires the lock; the other waits, then reads the slot as occupied, and moves on to select the next available slot. Optimistic locking is preferable when contention is low because it avoids lock-wait queuing overhead."

## Flashcards

| Q | A |
|---|---|
| What is the key preprocessing step for LC 2009 and why is deduplication necessary? | Sort the array and remove duplicates into a new array. Deduplication is necessary because duplicate values can never both appear in a valid continuous array of distinct integers — each duplicate is guaranteed to require a replacement operation. |
| How do you detect duplicates in an array in O(n) time and O(1) extra space (LC 442)? | For each element num, go to index abs(num)-1 and negate the value there. If it is already negative, abs(num) is a duplicate. This uses the array itself as a visited hash map via sign flipping. |
| In LC 1498, why do you add pow(2, right-left) for each valid left pointer? | When nums[left]+nums[right]<=target, every subset of the elements between left+1 and right (inclusive) forms a valid subsequence with nums[left] as the minimum. There are 2^(right-left) such subsets. |
| What is the Strategy design pattern and how does FeeCalculator use it? | Strategy encapsulates a family of algorithms behind a common interface and makes them interchangeable at runtime. FeeCalculator is the interface; HourlyFeeCalculator and FlatFeeCalculator are concrete strategies; ParkingLot holds a FeeCalculator reference and calls calculate(ticket) without knowing the pricing algorithm. |
| How does optimistic locking prevent a double-booking in a parking lot without holding a long database lock? | Each ParkingSlot row has a version integer. The UPDATE includes WHERE version=currentVersion. If another transaction committed first, the version has incremented and 0 rows are updated, signaling a conflict. The application then retries with the next available slot — no lock is held during the retry. |
