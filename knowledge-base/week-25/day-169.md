## DSA (1.5 hours — OA Contest Mode)

### Pattern: Memoized Game/Combinatorial DFS (State Enumeration)
- Core idea: When a problem requires simulating all possible outcomes across rounds or turns, encode the essential state compactly and memoize on it. Enumerate all legal transitions from each state (e.g., all ways opponents can lose in a round), recurse into each resulting state, and aggregate results (min/max of earliest/latest meeting round). Normalize symmetric states (e.g., `min(first, n+1-second)`) to reduce state space.
- Trigger condition: "Earliest and latest round where two specific players can meet" or any tournament/game simulation where you must enumerate all bracket outcomes — memoized DFS on (remaining_players, pos1, pos2).
- Time complexity: O(N^3) states, O(N) transitions per state → O(N^4) worst case, acceptable for N≤28 | Space complexity: O(N^3)

### Contest Template (write from memory before solving):

// Memoized game simulation: state = (n players remaining, pos1, pos2)
// Normalized: first < second always; use n+1-pos to fold mirror symmetry
private Map<String, int[]> memo = new HashMap<>();

private int[] solve(int n, int first, int second) {
    // normalize: reflect if first + second > n + 1
    if (first + second > n + 1) return solve(n, n + 1 - second, n + 1 - first);
    String key = n + "," + first + "," + second;
    if (memo.containsKey(key)) return memo.get(key);

    if (first + second == n + 1) {
        // they meet this round
        return memo.computeIfAbsent(key, k -> new int[]{round, round});
        // (pass round as parameter or compute from outer context)
    }

    int earliest = Integer.MAX_VALUE, latest = Integer.MIN_VALUE;
    // enumerate all ways the n - (n/2)*2 middle players (non-target matchups) can lose
    // track how positions of first and second evolve → recurse(n/2 or ceil(n/2), newFirst, newSecond)
    // ...
    return memo.put(key, new int[]{earliest, latest});
}

### Code Skeleton

```java
class Solution {

    // LC 1900 — The Earliest and Latest Rounds Where Players Compete
    // (cold solve — 30-min timer, narrate approach before opening IDE)
    // TODO: public int[] earliestAndLatest(int n, int firstPlayer, int secondPlayer)
    // Key: encode state as (n, first, second) where first < second (normalized).
    //      Base: if first + second == n + 1, they are paired → record current round.
    //      Transition: enumerate how many of the non-target matches go to each outcome;
    //      new positions of first/second in round+1 depend on how many opponents
    //      before them win/lose. Memoize on (n, first, second, round) or restructure
    //      so round is implicit in the recursion depth.
    private int earliest, latest;

    public int[] earliestAndLatest(int n, int firstPlayer, int secondPlayer) {
        earliest = Integer.MAX_VALUE;
        latest = Integer.MIN_VALUE;
        dfsGame(n, firstPlayer, secondPlayer, 1);
        return new int[]{earliest, latest};
    }

    private void dfsGame(int n, int f, int s, int round) {
        // normalize: ensure f < s; fold mirror symmetry
        if (f > s) { int tmp = f; f = s; s = tmp; }
        if (f + s == n + 1) {
            // they meet in this round
            earliest = Math.min(earliest, round);
            latest = Math.max(latest, round);
            return;
        }
        // Enumerate all possible outcomes of non-target matchups
        // Left segment: f-1 matchups before first player (players 1..f-1 vs players n..n-f+2)
        // Right segment: s-f-1 players between first and second
        // Key loops: try all k1 (winners from left of f) and k2 (winners between f and s)
        int half = n / 2;
        for (int k1 = 0; k1 <= f - 1; k1++) {
            // k1 = number of players from [1..f-1] who WIN (advance ahead of first)
            for (int k2 = 0; k2 <= s - f - 1; k2++) {
                // k2 = number of players between first and second who advance
                int newF = k1 + 1;
                int newS = k1 + k2 + 2;
                int newN = (n + 1) / 2; // ceil: survivor count in next round
                dfsGame(newN, newF, newS, round + 1);
            }
        }
    }

    // LC 1079 — Letter Tile Possibilities (revision — write from memory)
    public int numTilePossibilities(String tiles) {
        int[] freq = new int[26];
        for (char c : tiles.toCharArray()) freq[c - 'a']++;
        return backtrack(freq);
    }

    private int backtrack(int[] freq) {
        int count = 0;
        for (int i = 0; i < 26; i++) {
            if (freq[i] == 0) continue;
            freq[i]--;
            count += 1 + backtrack(freq); // place this char + all extensions
            freq[i]++;
        }
        return count;
    }

    // LC 526 — Beautiful Arrangement (OA simulation — set 20-min timer before reading)
    // TODO: public int countArrangement(int n)
    // Key (unlock after 20 min if stuck): backtrack with boolean[] used; at position i,
    //     try all unused j where j % i == 0 || i % j == 0; recurse to i+1.
    private int count526;

    public int countArrangement(int n) {
        count526 = 0;
        backtrack526(n, 1, new boolean[n + 1]);
        return count526;
    }

    private void backtrack526(int n, int pos, boolean[] used) {
        if (pos > n) { count526++; return; }
        for (int j = 1; j <= n; j++) {
            if (!used[j] && (j % pos == 0 || pos % j == 0)) {
                used[j] = true;
                backtrack526(n, pos + 1, used);
                used[j] = false;
            }
        }
    }
}
```

#### State Machine per Elevator

```
States: IDLE, MOVING_UP, MOVING_DOWN, AT_FLOOR, DOOR_OPEN, DOOR_CLOSED

Transitions:
  IDLE          → MOVING_UP      (upRequests non-empty, start heading up)
  IDLE          → MOVING_DOWN    (downRequests non-empty, start heading down)
  MOVING_UP     → AT_FLOOR       (currentFloor == upRequests.ceiling())
  MOVING_DOWN   → AT_FLOOR       (currentFloor == downRequests.floor())
  AT_FLOOR      → DOOR_OPEN      (automatic, 2-tick timer starts)
  DOOR_OPEN     → DOOR_CLOSED    (timer expires)
  DOOR_CLOSED   → MOVING_UP      (upRequests still non-empty)
  DOOR_CLOSED   → MOVING_DOWN    (only downRequests remain)
  DOOR_CLOSED   → IDLE           (both sets empty)
  MOVING_UP     → MOVING_DOWN    (upRequests exhausted, downRequests non-empty — LOOK reversal)
  MOVING_DOWN   → MOVING_UP      (downRequests exhausted, upRequests non-empty — LOOK reversal)
```

#### Scheduling Algorithms Compared

```
FCFS (First Come First Served):
  - Serve requests in arrival order regardless of position
  - Worst case: elevator ping-pongs across all floors
  - Simple to implement; O(1) scheduling

SCAN (Elevator Algorithm):
  - Move in one direction servicing all requests until reaching physical end (top/bottom)
  - Reverse direction at floor 0 or floor N
  - More efficient than FCFS; can cause starvation at extremes

LOOK:
  - Like SCAN but reverse at the LAST request in current direction (not physical end)
  - TreeSet.ceiling() / TreeSet.floor() implement this naturally
  - Best practical performance; implemented in Elevator.nextStop() above
```

**Interview talking point:** "If asked how to minimize average wait time in an elevator system with 4 elevators and 50 floors, answer: use the LOOK algorithm per elevator, and assign incoming requests to the elevator with the lowest effective distance score — current travel distance to the request floor plus a penalty if the elevator would need to reverse direction. This is similar to how disk scheduling minimizes seek time."

## Flashcards

| Q | A |
|---|---|
| In LC 1900, what is the base case and what does it mean? | `first + second == n + 1` — the two target players are matched against each other in this round (seats 1..n are paired as 1 vs n, 2 vs n-1, …). Record the current round number in both earliest and latest. |
| Why use `TreeSet<Integer>` for elevator requests instead of a `PriorityQueue`? | `TreeSet` supports `ceiling(x)` (next floor at or above x) and `floor(x)` (next floor at or below x) in O(log N) — exactly what LOOK algorithm needs to find the next stop in the current direction. `PriorityQueue` only gives the global minimum. |
| In LC 1079 (Letter Tile Possibilities), why does the frequency array approach automatically avoid counting duplicate sequences? | Because we iterate over distinct characters (freq[0..25]), not individual tile positions. Choosing the same character twice in a row is naturally handled by decrementing and restoring frequency — identical characters are treated as interchangeable. |
| What is the LOOK algorithm and how does it differ from SCAN? | LOOK reverses direction at the last pending request in the current direction (not at the physical floor boundary). SCAN always goes to floor 0 or floor N before reversing. LOOK eliminates empty travel past the last request. |
| In LC 526 (Beautiful Arrangement), what is the pruning condition that makes backtracking feasible? | At position i, only place values j where `j % i == 0 || i % j == 0`. This eliminates most candidates early, making the effective branching factor much lower than N — the actual count is at most a few hundred for N≤15. |
