## DSA (1.5 hours)

### Pattern: Sliding Window DP (Constrained Subsequence Sum)
- For LC 1425, the recurrence is `dp[i] = nums[i] + max(0, max(dp[i-k..i-1]))`. A naive O(nk) DP is too slow. Use a monotonic deque to maintain the maximum of the last k dp values in O(1) per step — same deque technique as Sliding Window Maximum (LC 239), but applied to DP values rather than raw array values.
- Trigger condition: "DP recurrence that depends on the maximum over a sliding window of previous dp values."
- Time complexity: O(n) | Space complexity: O(n) for dp array, O(k) for deque

### Pattern: Score-Defined Subarray Count with Two Pointers
- For LC 2302, a subarray's "score" = sum × length. The key insight: as the right pointer advances (window grows), the score only increases if we keep expanding — so for a fixed right, use a left pointer to find the maximum window where score < k. The count of valid subarrays ending at right = right - left + 1.
- Trigger condition: "Count subarrays where (sum × length) or similar multiplicative score satisfies a bound; score is monotone as window grows."
- Time complexity: O(n) | Space complexity: O(1)

### Problems

| # | Problem | LC # | Difficulty | Pattern | Key Insight |
|---|---------|------|------------|---------|-------------|
| 1 | Constrained Subsequence Sum | 1425 | Hard | Sliding Window DP + Monotonic Deque | `dp[i] = nums[i] + max(0, deque_max)`; deque stores indices of max dp values in last k positions |
| 2 | Count Subarrays With Score Less Than K | 2302 | Hard | Two Pointer + Running Sum | For each right, shrink left while `sum * length >= k`; valid count = right - left + 1 |
| 3 | Subarray Sum Equals K | 560 | Medium (Revision) | Prefix Sum + Hash Map | REVISION: `prefix[j] - prefix[i] == k`; count occurrences of `prefix - k` seen so far using a map |

### Code Skeleton

```python
# LC 1425 — Constrained Subsequence Sum
from collections import deque

def constrainedSubsetSum(nums: list, k: int) -> int:
    n = len(nums)
    dp = nums[:]  # dp[i] = max subsequence sum ending at i (must include nums[i])
    dq = deque()  # indices of max dp values in window [i-k, i-1]
    result = max(nums)  # answer could be a single negative element

    for i in range(n):
        # dp[i] = nums[i] + max(0, best dp value in last k positions)
        if dq:
            dp[i] = nums[i] + max(0, dp[dq[0]])

        result = max(result, dp[i])

        # Maintain deque: decreasing dp values, window of size k
        while dq and dp[dq[-1]] <= dp[i]:
            dq.pop()
        dq.append(i)
        if dq[0] == i - k + 1:  # TODO: check correct expiry condition
            dq.popleft()

    return result


# LC 2302 — Count Subarrays With Score Less Than K
def countSubarrays(nums: list, k: int) -> int:
    result = 0
    current_sum = 0
    left = 0

    for right in range(len(nums)):
        current_sum += nums[right]
        # score = current_sum * (right - left + 1)
        while current_sum * (right - left + 1) >= k:
            current_sum -= nums[left]
            left += 1
        result += right - left + 1  # all subarrays ending at right with valid left

    return result
```

Functional:

**Step 3 — Deep Dives (pick 2 in a real interview)**

**Deep Dive A — Tweet Write + Fan-out:**
1. Client POSTs tweet → API Gateway → Tweet Service
2. Tweet validated, Snowflake ID assigned, persisted to PostgreSQL (sharded by user_id)
3. Media upload handled separately: presigned S3 URL issued, async transcoding pipeline triggered
4. Kafka event `tweet.created` published
5. Fan-out Worker consumes event: if `follower_count < 50k` → push tweet_id to Redis `LPUSH user:{follower}:timeline` for each follower; `LTRIM` to 800 entries; if `follower_count >= 50k` → skip push (celebrities use pull-at-read-time)
6. Timeline Service on read: fetches Redis list (cached tweet_ids) + merges celebrity tweets (DB query) → hydrates from tweet cache → returns feed

**Deep Dive B — Rate Limiting:**
1. Rate limit middleware sits at API Gateway
2. Redis token bucket: `GET user:{id}:rate_limit` → check tokens remaining → `DECR` → if 0, return HTTP 429 with `Retry-After`
3. Sliding window for precision: `ZADD`, `ZREMRANGEBYSCORE`, `ZCARD` (atomic via Lua script)
4. Fail open if Redis is down (allow request, log for audit) vs fail closed (reject) — Twitter fails open for read endpoints, closed for high-abuse-risk write endpoints
