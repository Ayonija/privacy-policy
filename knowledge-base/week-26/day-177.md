## DSA (1.5 hours — Weak Area Targeted Drill)

### Pattern: Trie with Aggregate Node Statistics
- Core idea: Augment each TrieNode with a passCount field incremented for every word inserted through that node. This means passCount at depth d for a given prefix = number of words sharing that prefix of length d. Querying a word's prefix score then becomes a simple traversal summing passCount values — O(L) per query after O(total_chars) build.
- Trigger condition: "How many words share a given prefix?" or "rank autocomplete suggestions by frequency" — whenever you need aggregate counts per prefix level, not just presence/absence.
- Time complexity: O(total_chars) build, O(L) per query | Space complexity: O(total_chars × 26)

### Template (write from memory before solving):
```java
// Trie with pass count
class TrieNode {
    TrieNode[] children = new TrieNode[26];
    int passCount = 0; // how many words pass through this node
}
// Insert: for each char, create node if absent, increment passCount
// Score query: for each char in word, sum passCount at each node visited
```

### Problems
| # | Problem | LC # | Difficulty | Pattern | Key Insight |

| 1 | Sum of Prefix Scores of Strings | 2416 | Hard (Cold Solve, 30-min timer) | Trie + passCount | Build Trie; at each node, store passCount = how many words pass through this node; for each word, sum passCount of every node visited while inserting (those are the prefix scores) |
| 2 | Replace Words | 648 | Medium (Revision) | Trie + isEnd flag | REVISION: Build Trie from dictionary roots; for each word in sentence, traverse Trie char by char; if end-of-word encountered early → replace word with that prefix; else keep original word |
| 3 | Short Encoding of Words | 820 | Medium (OA Timed) | Trie of reversed words | OA: A word needs encoding only if it is NOT a suffix of another word. Build Trie of reversed words; only words that end at a Trie leaf (not extending further) need independent encoding; answer = sum(leaf_word_length + 1) |

### Code Skeleton
```java
class Solution {

    // LC 2416 — Sum of Prefix Scores of Strings (cold solve — 30-min timer)
    // TODO: public int[] sumPrefixScores(String[] words)
    // Key: build Trie incrementing passCount at every node during insert;
    //      for each word, query by summing passCount along the word's path.
    class TrieNode2416 {
        TrieNode2416[] ch = new TrieNode2416[26];
        int passCount = 0;
    }

    public int[] sumPrefixScores(String[] words) {
        TrieNode2416 root = new TrieNode2416();
        // Insert all words
        for (String w : words) {
            TrieNode2416 node = root;
            for (char c : w.toCharArray()) {
                int idx = c - 'a';
                if (node.ch[idx] == null) node.ch[idx] = new TrieNode2416();
                node = node.ch[idx];
                node.passCount++;
            }
        }
        // Score each word
        int[] ans = new int[words.length];
        for (int i = 0; i < words.length; i++) {
            TrieNode2416 node = root;
            int score = 0;
            for (char c : words[i].toCharArray()) {
                node = node.ch[c - 'a'];
                score += node.passCount;
            }
            ans[i] = score;
        }
        return ans;
    }

    // LC 648 — Replace Words (revision — write from memory)
    class TrieNode648 {
        TrieNode648[] ch = new TrieNode648[26];
        boolean isEnd = false;
    }

    public String replaceWords(List<String> dictionary, String sentence) {
        TrieNode648 root = new TrieNode648();
        for (String root648 : dictionary) {
            TrieNode648 node = root;
            for (char c : root648.toCharArray()) {
                int idx = c - 'a';
                if (node.ch[idx] == null) node.ch[idx] = new TrieNode648();
                node = node.ch[idx];
            }
            node.isEnd = true;
        }
        StringBuilder sb = new StringBuilder();
        for (String word : sentence.split(" ")) {
            if (sb.length() > 0) sb.append(' ');
            TrieNode648 node = root;
            StringBuilder prefix = new StringBuilder();
            boolean replaced = false;
            for (char c : word.toCharArray()) {
                if (node.ch[c - 'a'] == null) break;
                node = node.ch[c - 'a'];
                prefix.append(c);
                if (node.isEnd) { sb.append(prefix); replaced = true; break; }
            }
            if (!replaced) sb.append(word);
        }
        return sb.toString();
    }

    // LC 820 — Short Encoding of Words (OA timed — set 20-min timer before reading)
    // TODO: public int minimumLengthEncoding(String[] words)
    // Key (unlock after 20 min): reverse each word, insert into Trie; only words whose
    //     reversed form ends at a Trie leaf need independent encoding;
    //     answer = sum(len(word) + 1) for all leaf-ending words.
    class TrieNode820 {
        TrieNode820[] ch = new TrieNode820[26];
    }

    public int minimumLengthEncoding(String[] words) {
        TrieNode820 root = new TrieNode820();
        Map<TrieNode820, Integer> nodeToLen = new HashMap<>();
        for (String w : words) {
            TrieNode820 node = root;
            // insert reversed word
            for (int i = w.length() - 1; i >= 0; i--) {
                int idx = w.charAt(i) - 'a';
                if (node.ch[idx] == null) node.ch[idx] = new TrieNode820();
                node = node.ch[idx];
            }
            nodeToLen.put(node, w.length()); // map leaf node to word length
        }
        int ans = 0;
        for (Map.Entry<TrieNode820, Integer> e : nodeToLen.entrySet()) {
            TrieNode820 node = e.getKey();
            // leaf = no children → needs its own encoding slot
            boolean isLeaf = true;
            for (TrieNode820 child : node.ch) if (child != null) { isLeaf = false; break; }
            if (isLeaf) ans += e.getValue() + 1; // word length + '#' separator
        }
        return ans;
    }
}
```

## System Design (30 min — End-to-End Mock)

### Topic: Spotify — End-to-End Design

```
Scale: 600M users, 100M songs, 80M daily active, 30B streams/day

Core Services:
1. Music Catalog: PostgreSQL for metadata (track_id, title, artist_id, album_id, duration, genre, language); S3 for audio files (mp3/ogg at 96/160/320 kbps); CDN for delivery
2. Streaming: HLS (HTTP Live Streaming) — audio segmented into 10s chunks; client buffers 3 segments ahead; adaptive quality based on bandwidth; presigned S3 URL per chunk, expires in 1h
3. Playlist Service: Cassandra (user_id, playlist_id, track_list[]) for user playlists; PostgreSQL for curated playlists (editorial)
4. Recommendation:
   - Collaborative Filtering: matrix factorization (NMF) on listening matrix; weekly Spark batch job on billions of events; generates user and item vectors
   - Audio Analysis: CNN on spectrogram → tempo, key, energy, danceability vectors
   - ANN search: Annoy (Approximate Nearest Neighbors) finds similar users/tracks in <10ms
   - Discover Weekly: 30 songs → top 100 unseen songs from similar users' top plays
5. Offline Mode: premium feature; encrypted audio blobs downloaded to device; DRM key tied to Spotify account token; key validation on play (online check every 30 days)
6. Podcast Streaming: separate pipeline from music; RSS ingestion → transcoding → same CDN delivery

Key numbers:
- 30B streams/day ÷ 86400s ≈ 347K streams/second
- Audio file: 320kbps × 3.5min avg = ~8MB per song; 100M songs × 8MB = 800TB catalog
- CDN offloads 99%+ of audio delivery; origin handles only new content and cache misses
```

**Interview talking point:** "If asked how Spotify generates Discover Weekly for 500M users every Monday, answer: it's a batch job running on Apache Spark — Saturday/Sunday Spark processes 500B+ listening events to update user and song embedding vectors using matrix factorization. Monday morning, 500M personalized playlists are generated by running ANN (Approximate Nearest Neighbor) search to find the top unseen songs from users with similar taste vectors. The result is pre-computed and cached in DynamoDB — Discover Weekly is read from cache, not computed on-demand."

## Flashcards

| Q | A |
|---|---|
| How does the passCount field in a Trie enable LC 2416 Sum of Prefix Scores? | passCount at a node = number of words that have this node's prefix. When scoring a word, sum the passCount of every node traversed during lookup — each node's passCount is the number of words sharing that prefix length, which is the prefix score contribution. |
| In LC 648 Replace Words, how do you find the shortest root for a word using a Trie? | Insert all roots into Trie with isEnd=true at root's final char. For each word in sentence, traverse Trie char by char; the first node with isEnd=true encountered is the shortest matching root → replace word with that prefix. |
| In LC 820 Short Encoding of Words, which words can share an encoding? | A word can share encoding if it is a suffix of another word (it will appear naturally before a '#' of the longer word). Only words that are NOT a suffix of any other word need their own entry. Reverse all words, build Trie — leaf nodes identify words needing independent encoding. |
| How does Spotify achieve sub-10ms recommendation lookups for 500M users? | Pre-computation: embeddings and playlists are batch-generated weekly in Spark and stored in DynamoDB. At read time, Discover Weekly is a simple DynamoDB key-value lookup (user_id → playlist) — no real-time computation. |
| What is HLS and how does Spotify use it for adaptive streaming? | HTTP Live Streaming: audio is segmented into 10-second .ts or .aac chunks; a manifest file lists all segments and available bitrates. The client requests the manifest, then downloads segments one at a time based on current bandwidth — switching to lower bitrate chunks automatically when network degrades. |
