## LLD: Creational Patterns Quick Recall

| Pattern | One-liner | Machine coding trigger |
|---|---|---|
| **Singleton** | Ensures exactly one instance; provides global access point | "There should be only one X" |
| **Factory Method** | Delegates object creation; caller never calls `new ConcreteClass()` | "Create the right type of X based on input" |
| **Abstract Factory** | Creates families of compatible objects; one factory per product family | "Create a set of compatible X, Y, Z together" |
| **Builder** | Assembles one complex object step by step via fluent chaining | "Build a configurable X with many optional parts" |
| **Prototype** | Clone an existing object instead of constructing from scratch | "Create a new X that looks like this existing X" |

### Prototype Pattern (Brief)
**What it does:** Creates new objects by copying (cloning) an existing object — useful when construction is expensive or complex.

**Analogy:** Ctrl+C → Ctrl+V. You duplicate a document and then modify the copy; you don't rebuild the document from scratch.

**Code sketch:**
```java
interface Prototype {
    Prototype clone();
}

class Document implements Prototype {
    private String content;
    private List<String> styles;

    Document(Document other) {           // copy constructor
        this.content = other.content;
        this.styles  = new ArrayList<>(other.styles); // deep copy
    }

    public Prototype clone() { return new Document(this); }
}

// Usage
Document template = getExpensiveTemplate();
Document copy = (Document) template.clone(); // fast — no re-fetching
```

**When to use:** Object construction involves expensive DB calls or network fetches; you need many similar objects with small variations.

## Machine Coding: 4-Step Method

Before writing a single line of code:

1. **Nouns → Classes:** Extract entities from the problem statement. Each noun that has state and behavior is a candidate class.
2. **Verbs → Methods:** Extract actions. Each verb that operates on an entity becomes a method on the relevant class.
3. **What varies → Pattern:** Identify the axis of variability. Object creation varies? → Factory. Behavior varies? → Strategy. Count must be one? → Singleton. Construction is complex? → Builder.
4. **Start simple, extend:** Implement the happy path first. Add edge cases after the core model works.

## Machine Coding: Parking Lot — Full Design

### Problem Statement
Design a parking lot system. The lot has multiple levels, each with spots of different sizes. Vehicles park and unpark. The system issues tickets, calculates fees, and tracks availability.

---

### Step 1 — Nouns → Classes

| Entity | Responsibility |
|---|---|
| `ParkingLot` | The entire facility; entry point for all operations |
| `Level` | One floor of the lot; contains parking spots |
| `ParkingSpot` | A single spot; knows its size and whether it's occupied |
| `Vehicle` | Abstract parent; knows size and plate |
| `Car`, `Truck`, `Motorcycle` | Concrete vehicle types; extend Vehicle |
| `Ticket` | Issued on entry; holds spot, vehicle, entry time |
| `ParkingAttendant` | Coordinates park/unpark operations on a level |

---

### Step 2 — Verbs → Methods

| Method | On class | Description |
|---|---|---|
| `park(vehicle)` | `ParkingLot` | Find available spot → issue ticket |
| `unpark(ticket)` | `ParkingLot` | Free the spot → calculate fee |
| `getAvailableSpots(size)` | `Level` | Returns list of unoccupied spots matching size |
| `occupy(vehicle)` | `ParkingSpot` | Mark spot as taken |
| `free()` | `ParkingSpot` | Mark spot as available |
| `calculateFee(ticket)` | `FeeCalculator` | Duration × rate per pricing strategy |

---

### Step 3 — What Varies → Patterns

- **Vehicle type creation** varies by input string ("CAR", "TRUCK") → **Factory Method**
- **Pricing logic** varies (hourly, flat rate, premium) → **Strategy Pattern**
- **The ParkingLot itself** must be one instance → **Singleton**

### Core Code Skeleton

```java
// Singleton ParkingLot
class ParkingLot {
    private static volatile ParkingLot instance;
    private List<Level> levels;
    private FeeCalculator feeCalculator;

    private ParkingLot() {
        levels = new ArrayList<>();
        feeCalculator = new HourlyFeeCalculator(); // default strategy
    }

    public static ParkingLot getInstance() {
        if (instance == null) {
            synchronized (ParkingLot.class) {
                if (instance == null) instance = new ParkingLot();
            }
        }
        return instance;
    }

    public Ticket park(Vehicle vehicle) {
        for (Level level : levels) {
            Optional<ParkingSpot> spot = level.getAvailableSpot(vehicle.getSize());
            if (spot.isPresent()) {
                spot.get().occupy(vehicle);
                return new Ticket(spot.get(), vehicle, LocalDateTime.now());
            }
        }
        throw new ParkingLotFullException("No spot available for " + vehicle.getSize());
    }

    public double unpark(Ticket ticket) {
        ticket.setExitTime(LocalDateTime.now());
        ticket.getSpot().free();
        return feeCalculator.calculate(ticket);
    }
}

// Factory for vehicles
class VehicleFactory {
    public static Vehicle create(String type, String plate) {
        switch (type.toUpperCase()) {
            case "CAR":        return new Car(plate);
            case "TRUCK":      return new Truck(plate);
            case "MOTORCYCLE": return new Motorcycle(plate);
            default: throw new IllegalArgumentException("Unknown vehicle: " + type);
        }
    }
}

// Strategy: hourly fee
class HourlyFeeCalculator implements FeeCalculator {
    private static final double RATE_PER_HOUR = 50.0;

    public double calculate(Ticket ticket) {
        long minutes = ChronoUnit.MINUTES.between(ticket.getEntryTime(), ticket.getExitTime());
        return Math.ceil(minutes / 60.0) * RATE_PER_HOUR;
    }
}
```

---

### Key Methods Summary

| Method | Input | Output | Notes |
|---|---|---|---|
| `park(vehicle)` | Vehicle | Ticket | Searches levels in order; first-fit spot |
| `unpark(ticket)` | Ticket | double | Sets exit time; frees spot; delegates to strategy |
| `getAvailableSpots(size)` | VehicleSize | Optional<ParkingSpot> | Only returns spots that can fit the vehicle |
| `calculateFee(ticket)` | Ticket | double | Delegated to injected FeeCalculator strategy |

---

### Edge Cases

| Edge Case | Handling |
|---|---|
| Lot is full | `park()` throws `ParkingLotFullException` after exhausting all levels |
| Invalid ticket on unpark | Validate ticket not null; spot must be occupied by the same vehicle |
| Vehicle size > spot size | `getAvailableSpot()` only returns spots with `spotSize >= vehicleSize` |
| Same vehicle parks twice | Check `licensePlate` uniqueness on `park()` if required by problem |
| Concurrent access | ParkingSpot `occupy()` and `free()` methods synchronized; or use optimistic locking |

## Creational Confusion Table

| Symptom / smell | Reach for... | Why |
|---|---|---|
| "There should be only one X" | Singleton | Instance count control |
| "Create the right subtype of X based on input" | Factory Method | Type selection decoupled from caller |
| "Create a compatible set of X, Y, Z" | Abstract Factory | Product family consistency |
| "Object needs many optional config fields" | Builder | Telescoping constructor anti-pattern |
| "Construction is expensive; need a similar copy" | Prototype | Clone instead of rebuild |
