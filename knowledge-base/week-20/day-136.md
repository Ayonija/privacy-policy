# Day 136 — Graphs Hard: Directed Redundant Edge & Island Expansion
**Week 20 | Phase 3 — Mock Interviews | Month 5**

## Focus
Two structurally different hard graph problems — finding the extra edge in a directed graph that violates tree structure using Union-Find with parent analysis (LC 685) and maximizing island area by flipping one zero cell using Union-Find labeling (LC 827) — both test advanced graph reasoning under time pressure.

---

## DSA (1.5 hours)

### Pattern: Union-Find with Directed Edge Conflict Analysis
- For LC 685 (directed graph), there are two failure modes for the extra edge: (1) a node has two parents (in-degree 2), or (2) a cycle exists. If a node has two parents, it generates two candidate edges. Try removing candidate2 first; if the remaining graph has no cycle, return candidate2. Otherwise return candidate1. If no node has two parents, the extra edge is simply the one that creates a cycle (like LC 684 undirected version).
- Trigger condition: "Directed graph with n nodes and n edges — exactly one extra edge makes it not a valid rooted tree. Find the edge to remove."
- Time complexity: O(n α(n)) | Space complexity: O(n)

### Pattern: Union-Find Island Labeling + Neighbor Expansion
- For LC 827, label each island with Union-Find; store island sizes in a map keyed by root. For each zero cell, check its 4 neighbors — sum unique island sizes (deduplicate roots) + 1 (the flipped cell itself). Return the maximum such sum; if no zero cell exists, return total area.
- Trigger condition: "Maximize island size by flipping exactly one 0 to 1 — label islands, then for each 0 cell aggregate neighboring island sizes."
- Time complexity: O(n²) for n×n grid | Space complexity: O(n²)

### Code Skeleton

```java
class Solution {

    // LC 685 — Redundant Connection II
    public int[] findRedundantDirectedConnection(int[][] edges) {
        int n = edges.length;
        int[] parent = new int[n + 1];  // parent[i] = direct parent in the edge list
        int[] candidate1 = null, candidate2 = null;

        // Pass 1: find if any node has two parents
        for (int[] edge : edges) {
            int u = edge[0], v = edge[1];
            if (parent[v] != 0) {
                // v already has a parent — two candidate edges
                candidate1 = new int[]{parent[v], v};
                candidate2 = new int[]{u, v};
                // Skip candidate2 for now
            } else {
                parent[v] = u;
            }
        }

        // Pass 2: Union-Find cycle detection
        int[] uf = new int[n + 1];
        for (int i = 0; i <= n; i++) uf[i] = i;

        for (int[] edge : edges) {
            int u = edge[0], v = edge[1];
            if (candidate2 != null && edge[0] == candidate2[0] && edge[1] == candidate2[1]) {
                continue;  // skip candidate2 to test if graph is valid without it
            }
            int rootU = find(uf, u);
            int rootV = find(uf, v);
            if (rootU == rootV) {
                // Cycle detected — if no two-parent node, this edge is the answer
                return candidate1 == null ? edge : candidate1;
            }
            uf[rootU] = rootV;
        }
        return candidate2;  // removing candidate2 fixed the graph
    }

    private int find(int[] uf, int x) {
        if (uf[x] != x) uf[x] = find(uf, uf[x]);
        return uf[x];
    }


    // LC 827 — Making A Large Island
    private int[] ufParent, ufSize;

    public int largestIsland(int[][] grid) {
        int n = grid.length;
        ufParent = new int[n * n];
        ufSize = new int[n * n];
        Arrays.fill(ufSize, 1);
        for (int i = 0; i < n * n; i++) ufParent[i] = i;

        int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};

        // Step 1: union all land cells
        for (int r = 0; r < n; r++) {
            for (int c = 0; c < n; c++) {
                if (grid[r][c] == 1) {
                    for (int[] d : dirs) {
                        int nr = r + d[0], nc = c + d[1];
                        if (nr >= 0 && nr < n && nc >= 0 && nc < n && grid[nr][nc] == 1) {
                            union(r * n + c, nr * n + nc);
                        }
                    }
                }
            }
        }

        // Step 2: for each 0 cell, compute max island if flipped
        int maxSize = 0;
        boolean hasZero = false;

        for (int r = 0; r < n; r++) {
            for (int c = 0; c < n; c++) {
                if (grid[r][c] == 0) {
                    hasZero = true;
                    Set<Integer> seen = new HashSet<>();
                    int size = 1;  // the flipped cell itself
                    for (int[] d : dirs) {
                        int nr = r + d[0], nc = c + d[1];
                        if (nr >= 0 && nr < n && nc >= 0 && nc < n && grid[nr][nc] == 1) {
                            int root = find2(nr * n + nc);
                            if (seen.add(root)) size += ufSize[root];
                        }
                    }
                    maxSize = Math.max(maxSize, size);
                }
            }
        }

        return hasZero ? maxSize : n * n;  // if no zero, entire grid is one island
    }

    private int find2(int x) {
        if (ufParent[x] != x) ufParent[x] = find2(ufParent[x]);
        return ufParent[x];
    }

    private void union(int x, int y) {
        int rx = find2(x), ry = find2(y);
        if (rx == ry) return;
        if (ufSize[rx] < ufSize[ry]) { int t = rx; rx = ry; ry = t; }
        ufParent[ry] = rx;
        ufSize[rx] += ufSize[ry];
    }
}
```

## System Design (1 hour)

### Topic: Uber / Lyft — ETA & Route Calculation

**Core components:**
- **ETA Service** — given pickup and destination coordinates, returns estimated time of arrival for driver to reach rider + trip duration
- **Map/Routing Service** — Uber uses its own mapping stack (built on OpenStreetMap); computes shortest time path using Dijkstra or A* on a road graph with live traffic weights
- **Traffic Layer** — road segments have base travel times; real-time traffic data from driver GPS traces, external traffic APIs, and historical patterns adjusts weights; refreshed every few minutes
- **Route Caching** — popular (origin_zone, destination_zone) pairs are precomputed and cached in Redis; cache keyed by zone pair + time bucket (15-min intervals); TTL = 5 min
- **Fallback** — if in-house routing is unavailable, fall back to Google Maps Distance Matrix API (higher cost, external dependency)
- **ETA for driver-to-pickup** — uses driver's current GPS location and the road graph; recalculated every 30s and pushed to rider app via WebSocket

**Key trade-offs:**
- **Precomputed routes vs real-time Dijkstra:** precomputed covers popular paths in O(1) but misses real-time conditions; Dijkstra handles arbitrary routes but takes 5-50ms per query at city scale — Uber uses precomputed for zone pairs + Dijkstra for edge cases
- **Contraction Hierarchies (CH) vs plain Dijkstra:** CH preprocesses the graph to add shortcut edges, reducing query time from O(E log V) to O(s log s) where s << E — Uber and Lyft use CH for sub-millisecond routing on large city graphs
- **ETA accuracy:** overclaiming vs underclaiming — Uber adds buffer to ETAs to meet expectations; ML model trained on historical actual vs predicted times adds calibration layer

**Interview talking point:** "If asked how Uber computes ETA for millions of concurrent trips in under 100ms, answer: Uber uses Contraction Hierarchies (CH) — a preprocessing step that adds hierarchical shortcut edges to the road graph. At query time, bidirectional Dijkstra on the contracted graph converges in O(s log s) where s is the number of 'important' nodes visited, typically 100-1000 vs millions in plain Dijkstra. Combined with route caching for popular zone pairs, most ETAs are served in 5-20ms."

| In LC 685, what are the two failure modes that can make a directed graph not a valid rooted tree? | (1) A node has two incoming edges (two parents); (2) a cycle exists with no node having two parents |
| In LC 685, why try removing candidate2 first? | Candidate2 is the later-appearing edge; if the problem has a two-parent node, the last edge pointing to it is candidate2; removing it and checking for a valid tree is the correct first test |
| In LC 827, why use a HashSet of roots (not nodes) when summing neighbor islands? | The same island may have multiple cells bordering the 0 cell; using the root ID deduplicates so you don't count the same island's size twice |
| In LC 827, why does `find(root)` need to be called on each neighbor rather than using the raw cell index? | Union-Find may have compressed paths — the root after path compression is the canonical representative; raw index may point to a non-root node with a different (smaller) size |
